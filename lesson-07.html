الجزء الأول: الانتقالات (CSS Transitions)

Transitions تُحوّل التغييرات الفورية في CSS إلى تغييرات سلسة وجميلة. بدلاً من أن يتغيّر لون الزر فجأة عند hover، يتدرّج تدريجياً.

1.1

بنية transition

CSS 
/* الصيغة المختصرة */
/* transition: property duration timing-function delay; */

.btn {
  background-color: #2E75B6;
  transition: background-color 0.3s ease 0s;
  /*          الخاصية       المدة  التسهيل  التأخير */
}

.btn:hover {
  background-color: #1A3C5E;
  /* الآن التغيير يحدث بسلاسة خلال 0.3 ثانية */
}
1.2

خصائص transition التفصيلية

الخاصيةالوصف والقيم
transition-propertyالخاصية التي تُطبَّق عليها — all | color | transform | ...
transition-durationمدة الانتقال — 0.3s | 300ms | 1s
transition-timing-functionدالة التسهيل — ease, linear, ease-in, ease-out, ease-in-out, cubic-bezier()
transition-delayالتأخير قبل البدء — 0s | 0.2s | 500ms
CSS 
/* تفصيلي */
.element {
  transition-property: background-color, transform, box-shadow;
  transition-duration: 0.3s, 0.4s, 0.3s;
  transition-timing-function: ease, ease-out, ease;
  transition-delay: 0s, 0.1s, 0s;
}

/* مختصر — أكثر من خاصية */
.element {
  transition:
    background-color 0.3s ease,
    transform 0.4s ease-out 0.1s,
    box-shadow 0.3s ease;
}

/* all — يُطبَّق على كل الخصائص (استخدم بحذر) */
.element {
  transition: all 0.3s ease;
}
1.3

دوال التسهيل (Timing Functions / Easing)

دوال التسهيل تتحكم في سرعة الحركة خلال فترة الانتقال:

📈 دوال التسهيل — كيف تتصرف

ease: بطيء → سريع → بطيء (الأكثر استخداماً) ✅

linear: سرعة ثابتة طوال الانتقال

ease-in: بطيء → يتسارع (كالكرة تسقط)

ease-out: سريع → يتباطأ (كالكرة ترتطم بالأرض) ✅

ease-in-out: بطيء → سريع → بطيء (أكثر نعومة من ease)

cubic-bezier(x1, y1, x2, y2) — تحكم كامل مخصص

cubic-bezier(0.68, -0.55, 0.27, 1.55) → تأثير نابض (bounce)

cubic-bezier(0.4, 0, 0.2, 1) → Material Design ✅

cubic-bezier(0, 0, 0.2, 1) → تسارع ثم توقف ناعم

steps(4, end) → حركة مقطّعة (sprite animation)

steps(1) → تغيير فوري بدون انتقال

CSS 
/* استخدام cubic-bezier — أداة مفيدة: cubic-bezier.com */

/* زر ينبض عند hover */
.btn-bounce {
  transition: transform 0.4s cubic-bezier(0.68, -0.55, 0.27, 1.55);
}
.btn-bounce:hover { transform: scale(1.15); }

/* تأثير Material Design الرسمي */
.material-btn {
  transition: all 0.3s cubic-bezier(0.4, 0, 0.2, 1);
}

/* حركة مقطّعة (مثل الساعة العقارب) */
.clock-hand {
  transition: transform 1s steps(60, end);
}

الجزء الثاني: التحويلات (CSS Transforms)

transform تُغيّر شكل العنصر وموضعه وحجمه بصرياً دون التأثير على تدفق الصفحة. هي الخاصية المُفضّلة للتحريك لأنها تستخدم GPU وأداؤها ممتاز.

2.1

translate (الإزاحة)

CSS 
/* الإزاحة الأفقية والعمودية */
.element {
  transform: translateX(50px);    /* إزاحة أفقية 50px لليسار */
  transform: translateY(-20px);   /* إزاحة عمودية 20px للأعلى */
  transform: translate(50px, -20px); /* أفقي وعمودي معاً */

  /* بالنسبة المئوية — نسبة لحجم العنصر نفسه */
  transform: translateX(100%);   /* يتحرك بعرضه الكامل */
  transform: translateY(-100%);  /* يتحرك بارتفاعه للأعلى */

  /* 3D */
  transform: translateZ(50px);
  transform: translate3d(50px, 20px, 10px);
}

/* الاستخدام الأشهر: توسيط مطلق */
.centered {
  position: absolute;
  top: 50%; left: 50%;
  transform: translate(-50%, -50%); /* تراجع بنصف الحجم */
}
2.2

rotate (التدوير)

CSS 
.element {
  transform: rotate(45deg);      /* تدوير 45 درجة باتجاه عقارب الساعة */
  transform: rotate(-90deg);     /* تدوير 90 درجة عكس العقارب */
  transform: rotate(0.5turn);    /* نصف دورة = 180 درجة */
  transform: rotate(1turn);      /* دورة كاملة 360 درجة */

  /* 3D */
  transform: rotateX(45deg);     /* تدوير حول المحور X (يميل للخلف) */
  transform: rotateY(180deg);    /* تدوير حول المحور Y (ينقلب) */
  transform: rotateZ(30deg);     /* = rotate(30deg) */
}

/* مثال: زر القائمة الهامبرغر يتحول لـ X */
.hamburger { transition: transform 0.3s ease; }
.hamburger.open { transform: rotate(90deg); }

/* أيقونة سهم في قائمة منسدلة */
.arrow { transition: transform 0.3s ease; }
.open > .arrow { transform: rotate(180deg); }
2.3

scale (التكبير والتصغير)

CSS 
.element {
  transform: scale(1.2);         /* تكبير 120% في الاتجاهين */
  transform: scale(0.8);         /* تصغير إلى 80% */
  transform: scaleX(1.5);        /* تمديد أفقي فقط */
  transform: scaleY(0.5);        /* تنصيف عمودي فقط */
  transform: scale(1.2, 0.8);    /* تكبير أفقي + تصغير عمودي */
  transform: scale(-1);          /* انعكاس كامل (مرآة) */
  transform: scaleX(-1);         /* انعكاس أفقي فقط */
}

/* مثال: زر ينكمش عند الضغط */
.btn { transition: transform 0.15s ease; }
.btn:hover  { transform: scale(1.05); }
.btn:active { transform: scale(0.96); }

/* مثال: صورة تكبر عند hover */

.photo img { transition: transform 0.4s ease; }
.photo:hover img { transform: scale(1.1); }

دمج Transforms المتعددة

CSS 
/* دمج عدة transforms في تصريح واحد */
/* الترتيب مهم جداً! */
.element {
  /* translate أولاً ثم rotate */
  transform: translateX(100px) rotate(45deg);

  /* rotate أولاً ثم translate (نتيجة مختلفة!) */
  transform: rotate(45deg) translateX(100px);
}

/* مثال كامل: بطاقة منبثقة */
.popup-card {
  transition: transform 0.4s cubic-bezier(0.34, 1.56, 0.64, 1);
}
.popup-card:hover {
  transform: translateY(-12px) scale(1.03) rotate(-1deg);
}

/* تحكم في نقطة الأصل (pivot point) */
.element {
  transform-origin: center;      /* الافتراضي */
  transform-origin: top right;   /* من الزاوية العلوية اليمنى */
  transform-origin: 0% 50%;      /* من المنتصف الأيمن */
  transform-origin: 100px 50px;  /* نقطة مخصصة */
}

/* مثال: ورقة تُفتح */
.page {
  transform-origin: right center; /* تنقلب من الجهة اليمنى */
  transition: transform 0.6s ease;
}
.page:hover { transform: rotateY(-20deg); }

الجزء الثالث: الأنيميشن بـ @keyframes

@keyframes تُتيح تعريف تسلسل الحركة بدقة من خطوة إلى خطوة — أقوى بكثير من transition لأنها لا تحتاج تفاعل المستخدم.

3.1

تعريف Keyframes

CSS 
/* الطريقة 1: from و to (بداية ونهاية) */
@keyframes fadeIn {
  from { opacity: 0; }
  to   { opacity: 1; }
}

/* الطريقة 2: نسب مئوية (أكثر تحكماً) */
@keyframes slideInRight {
  0%   { transform: translateX(100px); opacity: 0; }
  60%  { transform: translateX(-15px); opacity: 1; }
  80%  { transform: translateX(8px); }
  100% { transform: translateX(0); }
}

/* نقاط متعددة بنفس التأثير */
@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50%       { transform: scale(1.1); }
}

/* تحريك اللون بتسلسل */
@keyframes colorCycle {
  0%   { background: #1A3C5E; }
  33%  { background: #2E75B6; }
  66%  { background: #E8A020; }
  100% { background: #1A3C5E; }
}
3.2

خصائص animation

الخاصيةالوصف والقيم الشائعة
animation-nameاسم الـ @keyframes المُعرَّف
animation-durationمدة دورة واحدة — 0.5s | 1s | 2000ms
animation-timing-functionدالة التسهيل — ease, linear, ease-in-out, cubic-bezier()
animation-delayتأخير قبل البدء — 0s | 0.5s | -1s (يبدأ من المنتصف)
animation-iteration-countعدد التكرار — 1 | 3 | infinite
animation-directionnormal | reverse | alternate | alternate-reverse
animation-fill-modenone | forwards | backwards | both — حالة العنصر قبل/بعد
animation-play-staterunning | paused — للتحكم بـ JavaScript
CSS 
/* الصيغة المختصرة */
/* animation: name duration timing delay iteration direction fill-mode; */

.element {
  animation: fadeIn 0.8s ease 0.2s 1 normal forwards;
/*           الاسم  المدة  التسهيل التأخير التكرار الاتجاه fill-mode */
}

/* أمثلة شائعة */
.fade-in     { animation: fadeIn 0.6s ease forwards; }
.pulse       { animation: pulse 1.5s ease-in-out infinite; }
.spin        { animation: spin 1s linear infinite; }
.bounce-in   { animation: bounceIn 0.8s cubic-bezier(0.34,1.56,0.64,1) forwards; }

/* أنيميشنات متعددة على نفس العنصر */
.special {
  animation:
    fadeIn 0.5s ease forwards,
    pulse 2s ease 0.5s infinite;
}
3.3

animation-fill-mode بالتفصيل

CSS 
@keyframes slideDown {
  from { transform: translateY(-30px); opacity: 0; }
  to   { transform: translateY(0);     opacity: 1; }
}

/* none: العنصر يعود لحالته الأصلية بعد الانتهاء */
.el { animation: slideDown 0.5s ease none; }

/* forwards: يبقى عند آخر نقطة (to/100%) بعد الانتهاء ✅ */
.el { animation: slideDown 0.5s ease forwards; }

/* backwards: يبدأ من أول نقطة حتى أثناء التأخير */
.el { animation: slideDown 0.5s ease 1s backwards; }

/* both: forwards + backwards معاً ✅ (الأفضل عادةً) */
.el { animation: slideDown 0.5s ease 1s both; }
3.4

animation-direction

CSS 
/* normal: من 0% → 100% (الافتراضي) */
.el { animation: slideIn 1s linear infinite normal; }

/* reverse: من 100% → 0% (معكوس) */
.el { animation: slideIn 1s linear infinite reverse; }

/* alternate: 0%→100% ثم 100%→0% بالتناوب ✅ */
.el { animation: pulse 1s ease infinite alternate; }

/* alternate-reverse: 100%→0% ثم 0%→100% */
.el { animation: pulse 1s ease infinite alternate-reverse; }

/* مثال حركة نبض مستمرة وناعمة */
@keyframes heartbeat {
  from { transform: scale(1); }
  to   { transform: scale(1.15); }
}
.heart {
  animation: heartbeat 0.6s ease-in-out infinite alternate;
}

الجزء الخامس: أداء الأنيميشن — أفضل الممارسات

5.1

الخصائص الآمنة للتحريك (GPU-Accelerated)

المتصفح يُفوّض تحريك بعض الخصائص للـ GPU مباشرة — هذه الخصائص لا تُسبّب إعادة حساب تخطيط الصفحة:

الخاصيةالاستخدام
transform: translate()✅ التحريك والإزاحة — استخدم بدلاً من top/left
transform: scale()✅ التكبير والتصغير — استخدم بدلاً من width/height
transform: rotate()✅ التدوير — آمن تماماً
opacity✅ الشفافية — آمن تماماً
filter: blur()✅ نسبياً — أحياناً يُسبّب ضغطاً
5.2

الخصائص المكلفة — تجنّبها في الأنيميشن

تجنّب تحريك هذه الخصائصالسبب
width, height❌ يُعيد حساب تخطيط الصفحة (Layout Reflow)
top, left, right, bottom❌ يُعيد حساب الموضع — استخدم transform بدلاً
margin, padding❌ يُؤثر على الـ Layout
font-size❌ يُعيد رسم النص والتخطيط
border-width❌ يُعيد حساب الـ Layout
box-shadow (بحذر)⚠️ مكلف — استخدم باعتدال أو filter: drop-shadow
5.3

will-change للتحسين المسبق

CSS 
/* will-change يُخبر المتصفح مسبقاً بما سيتغير */
/* فيُهيّئ طبقة GPU مسبقاً */

.card {
  will-change: transform, opacity;
}

/* استخدم will-change فقط على عناصر ستتحرك فعلاً */
/* لا تضعه على كل العناصر — يستهلك ذاكرة */

/* الأفضل: أضفه عند hover وأزله بعد الانتهاء بالجافاسكريبت */
.btn:hover { will-change: transform; }
.btn:not(:hover) { will-change: auto; }
