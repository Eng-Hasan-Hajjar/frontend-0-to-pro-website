الجلسة التاسعة عشرة — useEffect Hook والتعامل مع Side Effects

Side Effects • useEffect • Dependency Array • Cleanup Function • جلب البيانات (Data Fetching)

👤 المؤلف: Eng. Hasan Hajjar  ·  Software Engineer & AI Engineer

🎯 أهداف الجلسة التاسعة عشرة

  • فهم ما هو Side Effect ولماذا يحتاج React آلية خاصة للتعامل معه
  • استخدام useEffect الأساسي لتنفيذ كود بعد كل عملية رسم
  • التحكم بدقّة في متى يُعاد تشغيل التأثير عبر مصفوفة الاعتماديات (Dependency Array)
  • فهم الفروق العملية بين عدم وجود مصفوفة، مصفوفة فارغة، ومصفوفة تحوي قيماً
  • كتابة دالة تنظيف (Cleanup) صحيحة لمنع تسرّب الذاكرة (Memory Leaks)
  • استخدام useEffect لجلب بيانات من API خارجي مع حالات تحميل وخطأ
  • التعامل الصحيح مع async/await داخل useEffect
  • بناء تطبيق طقس كامل يجلب بيانات حيّة عند تحميل الصفحة
🔁 مراجعة سريعة من الجلسة الثامنة عشرة

State ذاكرة خاصة بالمكوّن، وأي تغيير فيها عبر setState يُعيد تصيير المكوّن

useState: const [value, setValue] = useState(initial)

استخدم الشكل الدالي setValue(prev => ...) عند الاعتماد على القيمة السابقة

لا تُعدِّل الكائنات والمصفوفات مباشرة — أنشئ دائماً نسخة جديدة (Immutability)

رفع الحالة للأعلى (Lifting State Up) يُشارك البيانات بين مكوّنات إخوة

🎓 أهلاً بعالم Side Effects!

حتى الآن، كل ما بنيناه كان "نقياً": بيانات تدخل (Props/State) وواجهة تخرج (JSX)، بلا أي تواصل مع العالم الخارجي.

لكن التطبيقات الحقيقية تحتاج التواصل مع الخارج: جلب بيانات من خادم، ضبط مؤقّت، الاستماع لحدث في المتصفح، تعديل عنوان الصفحة مباشرة.

هذه العمليات تُسمّى Side Effects، وuseEffect هو الأداة الرسمية في React للتعامل معها بأمان.

🌊

الجزء الأول: ما هو Side Effect؟

1.1

التعريف

Side Effect هو أي عملية "تتفاعل مع العالم خارج المكوّن نفسه" — بعكس حساب واجهة من Props وState فقط:

  • جلب بيانات من خادم (Fetch API)
  • ضبط مؤقّت (setTimeout / setInterval)
  • الاستماع لأحداث المتصفح (resize، scroll، keydown)
  • التعديل المباشر على DOM أو عنوان الصفحة (document.title)
  • الاشتراك/إلغاء الاشتراك في خدمات خارجية (WebSocket مثلاً)
1.2

لماذا لا نضع هذه العمليات مباشرة داخل المكوّن؟

JSX 
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);

  // ❌ خطأ فادح: fetch يُستدعى مباشرة أثناء تنفيذ دالة المكوّن!
  fetch(`https://api.example.com/users/${userId}`)
    .then(res => res.json())
    .then(data => setUser(data)); // setUser تُسبِّب إعادة رسم

  // إعادة الرسم تُنفِّذ الدالة بأكملها من جديد → fetch يُستدعى مجدداً
  // → setUser تُستدعى مجدداً → إعادة رسم أخرى → حلقة لا نهائية! 🔥

  return <div>{user?.name}</div>;
}
⚠️ المشكلة

أي عملية "جانبية" داخل جسم دالة المكوّن مباشرة تُنفَّذ في كل مرة يُعاد فيها الرسم — بلا سيطرة على التوقيت. useEffect يمنحنا هذه السيطرة بالضبط.

🎣

الجزء الثاني: useEffect الأساسي

2.1

Syntax الأساسي

JSX 
import { useEffect, useState } from 'react';

function Component() {
  useEffect(() => {
    // الكود هنا يعمل "بعد" اكتمال عملية الرسم، لا أثناءها
    console.log('تمّ رسم المكوّن!');
  });

  return <div>مرحباً</div>;
}
2.2

مثال: تحديث عنوان الصفحة

JSX 
function Counter() {
  const [count, setCount] = useState(0);

  useEffect(() => {
    document.title = `عدد النقرات: ${count}`;
  });

  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
// كل ضغطة → إعادة رسم → useEffect يعمل بعدها مباشرة → عنوان تبويب المتصفح يتحدَّث
📋

الجزء الثالث: مصفوفة الاعتماديات (Dependency Array)

الشكلمتى يعمل التأثير
useEffect(fn)بعد كل عملية رسم، بلا أي استثناء
useEffect(fn, [])مرة واحدة فقط، عند ظهور المكوّن لأول مرة (Mount)
useEffect(fn, [a, b])عند الظهور الأول، وكل مرة تتغيّر فيها a أو b فقط
3.1

الحالات الثلاث في الكود

React 
useEffect(() => {
  console.log('يعمل بعد كل رسم — بلا استثناء');
});

useEffect(() => {
  console.log('يعمل مرة واحدة فقط، عند ظهور المكوّن أول مرة');
}, []); // مصفوفة فارغة

useEffect(() => {
  console.log('يعمل عند الظهور الأول، وكل مرة تتغيّر فيها userId فقط');
}, [userId]); // مصفوفة تحوي قيمة
⭐ القاعدة الذهبية

ضع في المصفوفة كل قيمة (State أو Props) يستخدمها التأثير من الخارج. نسيان قيمة يُسبِّب أخطاءً منطقية صعبة الاكتشاف (Stale Closures).

🧹

الجزء الرابع: دالة التنظيف (Cleanup Function)

بعض التأثيرات (مؤقّتات، مستمعو أحداث، اشتراكات) تبقى تعمل في الخلفية حتى بعد اختفاء المكوّن، ما لم نُنظِّفها صراحة.

4.1

مؤقّت (Interval) مع تنظيف صحيح

JSX 
function Timer() {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const id = setInterval(() => {
      setSeconds(prev => prev + 1);
    }, 1000);

    // دالة التنظيف: تُنفَّذ عند اختفاء المكوّن، أو قبل إعادة تشغيل التأثير
    return () => clearInterval(id);
  }, []); // تشغيل مرة واحدة فقط

  return <p>⏱️ الوقت المنقضي: {seconds} ثانية</p>;
}
⚠️ بدون تنظيف

لو غادر المستخدم هذه الصفحة دون clearInterval، سيبقى المؤقّت يعمل في الخلفية للأبد — تسرّب ذاكرة حقيقي، وقد يُسبِّب أخطاءً إن حاول تحديث State لمكوّن لم يعد موجوداً.

4.2

الاستماع لحدث المتصفح مع إزالته

JSX 
function WindowWidth() {
  const [width, setWidth] = useState(window.innerWidth);

  useEffect(() => {
    function handleResize() {
      setWidth(window.innerWidth);
    }
    window.addEventListener('resize', handleResize);

    return () => window.removeEventListener('resize', handleResize);
  }, []);

  return <p>📐 عرض النافذة الحالي: {width}px</p>;
}

الجزء الخامس: جلب البيانات (Data Fetching) بـ useEffect

5.1

النمط الأساسي: حالات تحميل وخطأ

JSX 
function UserProfile({ userId }) {
  const [user, setUser] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    setLoading(true);
    setError(null);

    fetch(`https://api.example.com/users/${userId}`)
      .then(res => {
        if (!res.ok) throw new Error('فشل الطلب');
        return res.json();
      })
      .then(data => setUser(data))
      .catch(err => setError(err.message))
      .finally(() => setLoading(false));
  }, [userId]); // يُعاد الجلب فقط عند تغيّر userId

  if (loading) return <p>⏳ جارٍ التحميل...</p>;
  if (error) return <p>❌ خطأ: {error}</p>;
  return <div>{user.name}</div>;
}
5.2

التعامل مع async/await داخل useEffect

React 
// ❌ خطأ: useEffect لا يقبل دالة async مباشرة كوسيط أول
useEffect(async () => {
  const res = await fetch(url);
  // React يتوقّع من التأثير إما لا شيء أو دالة تنظيف — وليس Promise!
}, []);
React 
// ✅ الحل: عرِّف دالة async داخلية واستدعها فوراً
useEffect(() => {
  async function fetchData() {
    const res = await fetch(url);
    const data = await res.json();
    setData(data);
  }
  fetchData();
}, [url]);

الجزء السادس: المشروع — تطبيق طقس يجلب بيانات حيّة

تطبيق يجلب بيانات الطقس الفعلية من API مجاني بلا مفتاح (Open-Meteo)، مع حالات تحميل وخطأ، ويُعيد الجلب كلما تغيّرت المدينة.

src/App.jsx

JSX 
import { useState, useEffect } from 'react';

function App() {
  const [city, setCity] = useState('دمشق');
  const [weather, setWeather] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    setLoading(true);
    setError(null);

    fetch('https://api.open-meteo.com/v1/forecast?latitude=33.5&longitude=36.3&current_weather=true')
      .then(res => {
        if (!res.ok) throw new Error('تعذّر جلب بيانات الطقس');
        return res.json();
      })
      .then(data => setWeather(data.current_weather))
      .catch(err => setError(err.message))
      .finally(() => setLoading(false));
  }, [city]);

  return (
    <div className="weather-app">
      <h1>🌤️ تطبيق الطقس</h1>
      <input
        value={city}
        onChange={(e) => setCity(e.target.value)}
        placeholder="اسم المدينة..."
      />

      {loading && <p>⏳ جارٍ التحميل...</p>}
      {error && <p className="error">❌ {error}</p>}
      {weather && !loading && (
        <div className="weather-card">
          <h2>{city}</h2>
          <p className="temp">{weather.temperature}°C</p>
          <p>سرعة الرياح: {weather.windspeed} كم/س</p>
        </div>
      )}
    </div>
  );
}

export default App;

الجزء السابع: تمارين الجلسة التاسعة عشرة

✏️ تمرين 1 — عنوان صفحة ديناميكي15 دقيقة

  • أنشئ State لعدد "إشعارات غير مقروءة"
  • استخدم useEffect لتحديث document.title ليعرض العدد بين قوسين، مثل: (3) اسم التطبيق
  • أضف زر يزيد العدد ليتأكد الطالب أن العنوان يتحدَّث فوراً

✏️ تمرين 2 — Countdown Timer20 دقيقة

  • حقل إدخال لعدد الثواني، وزر "ابدأ"
  • عدّاد تنازلي يعمل بـ setInterval داخل useEffect
  • يتوقف تلقائياً عند الوصول للصفر، مع رسالة "انتهى الوقت! ⏰"
  • تأكد من استخدام دالة تنظيف صحيحة (clearInterval)

✏️ تمرين 3 — محرّك بحث حيّ25 دقيقة

  • مصفوفة أسماء طلاب (10 عناصر على الأقل) في State
  • حقل بحث يُصفِّي القائمة فوراً أثناء الكتابة
  • استخدم useEffect لمحاكاة "تأخير شبكة" بسيط (setTimeout 300ms) قبل عرض النتائج
  • اعرض "لا توجد نتائج" عند عدم وجود تطابق

ملخص الجلسة التاسعة عشرة

الموضوعأبرز ما تعلمناه
Side Effectأي عملية تتفاعل مع خارج المكوّن: شبكة، مؤقّت، DOM، أحداث المتصفح
useEffect الأساسييعمل بعد كل عملية رسم افتراضياً
مصفوفة الاعتماديات[] = مرة واحدة فقط، [قيم] = عند تغيّرها، بلا مصفوفة = كل رسم
Cleanup Functionreturn () => {...} لإلغاء المؤقّتات والمستمعين ومنع تسرّب الذاكرة
جلب البياناتfetch داخل useEffect مع حالات loading/error/data
async في useEffectعرِّف دالة async داخلية واستدعها، لا تجعل التأثير نفسه async
المشروعتطبيق طقس يجلب بيانات حيّة مع تحميل وخطأ ديناميكيين

الواجب: تطبيق أخبار حيّة بتحديث تلقائي

أنشئ تطبيق React بـ Vite يجلب مقالات من مصدر بيانات (يمكن استخدام JSONPlaceholder أو أي API مجاني) مع تحديث تلقائي دوري:

◆ المتطلبات الوظيفية:

  • جلب قائمة مقالات عند تحميل الصفحة (useEffect + [])
  • عرض حالة "جارٍ التحميل" أثناء الجلب الأول
  • زر "تحديث الآن" يُعيد الجلب يدوياً
  • تحديث تلقائي كل 60 ثانية عبر setInterval داخل useEffect
  • عرض رسالة خطأ واضحة عند فشل الاتصال

◆ متطلبات تقنية:

  • دالة تنظيف صحيحة تُلغي المؤقّت الدوري (clearInterval) عند اختفاء المكوّن
  • مصفوفة اعتماديات صحيحة في كل useEffect (بلا تحذيرات في الكونسول)
  • فصل منطق الجلب في دالة async داخلية منفصلة
  • تقسيم الواجهة لمكوّنين على الأقل: App وArticleCard
