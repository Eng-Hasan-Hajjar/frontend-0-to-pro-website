الجلسة العشرون — useRef وCustom Hooks

useRef • الوصول لـ DOM • useState مقابل useRef • بناء Custom Hooks • useFetch • useLocalStorage

👤 المؤلف: Eng. Hasan Hajjar  ·  Software Engineer & AI Engineer

🎯 أهداف الجلسة العشرين

  • فهم ما هو useRef ولماذا يختلف جذرياً عن useState
  • الوصول المباشر لعناصر DOM باستخدام useRef (كالتركيز التلقائي على حقل)
  • استخدام useRef لحفظ قيم تبقى بين عمليات إعادة الرسم دون التسبب برسم جديد
  • معرفة متى بالضبط نستخدم useState ومتى نستخدم useRef
  • بناء أول Custom Hook خاص بنا لاستخراج منطق متكرر
  • بناء useFetch Hook قابل لإعادة الاستخدام في أي مكوّن
  • بناء useLocalStorage Hook يُزامن الحالة تلقائياً مع التخزين المحلي
  • دمج عدة Custom Hooks معاً في مشروع واحد متكامل
🔁 مراجعة سريعة من الجلسة التاسعة عشرة

useEffect ينفِّذ كوداً بعد كل عملية رسم، ويُستخدَم للتأثيرات الجانبية (Side Effects)

مصفوفة الاعتماديات تتحكّم بتوقيت إعادة تشغيل التأثير: [] مرة واحدة، [قيم] عند تغيّرها

دالة التنظيف (return () => {...}) ضرورية لإلغاء المؤقّتات والمستمعين

جلب البيانات يحتاج ثلاث حالات: loading وerror وdata

لا نجعل دالة useEffect نفسها async — نُعرِّف دالة داخلية ونستدعيها

🎓 أهلاً بعالم إعادة الاستخدام!

لاحظت في الجلسات الماضية أننا كتبنا نفس منطق "التحميل/الخطأ/البيانات" أكثر من مرة؟ اليوم نحلّ هذا نهائياً.

سنتعلّم useRef أولاً — أداة تُشبه useState لكن بدون إعادة رسم، ثم نبني Custom Hooks: دوالنا الخاصة القابلة لإعادة الاستخدام في أي مكوّن.

🎯

الجزء الأول: useRef — الوصول المباشر لعناصر DOM

1.1

المشكلة: كيف نصل لعنصر HTML مباشرة؟

في JavaScript الخالص كنا نستخدم document.querySelector للوصول لعنصر. لكن React يدير DOM بنفسه، ولا يجوز التدخّل المباشر فيه بهذه الطريقة داخل مكوّن.

1.2

useRef الأساسي وسمة ref

JSX 
import { useRef } from 'react';

function TextInput() {
  const inputRef = useRef(null);

  function handleClick() {
    inputRef.current.focus(); // وصول مباشر وآمن للعنصر الحقيقي في DOM!
  }

  return (
    <div>
      <input ref={inputRef} type="text" />
      <button onClick={handleClick}>ركّز على الحقل</button>
    </div>
  );
}
💡 كيف تعمل

useRef(null) يُنشئ "صندوقاً" ثابتاً {current: null}. ربطه بسمة ref={inputRef} على عنصر JSX يجعل React يملأ inputRef.current بالعنصر الحقيقي بعد الرسم — دون أي إعادة رسم إضافية.

1.3

مثال: التركيز التلقائي عند التحميل

JSX 
function SearchBox() {
  const inputRef = useRef(null);

  useEffect(() => {
    inputRef.current.focus(); // يركّز تلقائياً فور ظهور المكوّن أول مرة
  }, []);

  return <input ref={inputRef} placeholder="ابحث هنا..." />;
}

الجزء الثاني: useRef لحفظ قيم بين الرسومات

المعيارuseStateuseRef
يُسبِّب إعادة رسم عند التغيير؟نعملا أبداً
طريقة القراءةالمتغيّر مباشرةعبر .current
تبقى القيمة محفوظة بين الرسومات؟نعمنعم
الاستخدام النموذجيبيانات تظهر في الواجهةالوصول لـ DOM، أو قيم لا تحتاج عرضاً
2.1

مثال: عدّاد إعادة الرسم

JSX 
function RenderCounter() {
  const [text, setText] = useState('');
  const renderCount = useRef(0);

  renderCount.current = renderCount.current + 1; // لا يُسبِّب إعادة رسم!

  return (
    <div>
      <input value={text} onChange={(e) => setText(e.target.value)} />
      <p>عدد مرات إعادة الرسم: {renderCount.current}</p>
    </div>
  );
}
⚠️ لماذا ليس useState هنا؟

لو استخدمنا useState لعدّاد الرسم، لدخلنا في حلقة لا نهائية: تحديث العدّاد → إعادة رسم → تحديث العدّاد مجدداً... useRef يتجنّب هذا تماماً لأنه لا "يُخبر" React بأي تغيير.

2.2

نمط القيمة السابقة (Previous Value)

JSX 
function usePrevious(value) {
  const ref = useRef();
  useEffect(() => {
    ref.current = value; // يُحفَظ "بعد" كل رسم، فيحمل القيمة السابقة خلال الرسم التالي
  });
  return ref.current;
}

function Counter() {
  const [count, setCount] = useState(0);
  const prevCount = usePrevious(count);

  return (
    <div>
      <p>الحالي: {count}، السابق: {prevCount ?? '—'}</p>
      <button onClick={() => setCount(count + 1)}>زيادة</button>
    </div>
  );
}

الجزء الثالث: متى نستخدم useState ومتى useRef؟

السؤالالإجابة
هل القيمة يجب أن تظهر في الواجهة (JSX)؟✅ استخدم useState
هل تغيّر القيمة يجب أن يُحدِّث الشاشة فوراً؟✅ استخدم useState
هل تحتاج فقط "تذكّر" قيمة بصمت دون عرضها؟🔧 استخدم useRef
هل تحتاج الوصول المباشر لعنصر DOM؟🔧 استخدم useRef

الجزء الرابع: بناء أول Custom Hook

Custom Hook هو دالة JavaScript عادية تبدأ اسمها بـ use، وتستخدم Hooks أخرى بداخلها لاستخراج منطق متكرر يمكن مشاركته بين عدة مكوّنات.

4.1

القاعدة الأساسية

  • الاسم يجب أن يبدأ بـ use (useToggle، useFetch...) — هذا يُخبر React وESLint أنها Hook فعلية
  • يمكنها استدعاء Hooks أخرى بداخلها (useState، useEffect، useRef...)
  • تخضع لنفس قواعد الـ Hooks: لا شروط، لا حلقات، القمة فقط
4.2

مثال بسيط: useToggle

React 
// hooks/useToggle.js
import { useState } from 'react';

function useToggle(initialValue = false) {
  const [value, setValue] = useState(initialValue);

  function toggle() {
    setValue((prev) => !prev);
  }

  return [value, toggle];
}

export default useToggle;
JSX 
// الاستخدام في أي مكوّن
function DarkModeButton() {
  const [isDark, toggleDark] = useToggle(false);

  return (
    <button onClick={toggleDark}>
      {isDark ? '🌙 داكن' : '☀️ فاتح'}
    </button>
  );
}

الجزء الخامس: useFetch — Custom Hook لجلب البيانات

5.1

بناء useFetch كامل

React 
// hooks/useFetch.js
import { useState, useEffect } from 'react';

function useFetch(url) {
  const [data, setData] = useState(null);
  const [loading, setLoading] = useState(true);
  const [error, setError] = useState(null);

  useEffect(() => {
    setLoading(true);
    setError(null);

    fetch(url)
      .then((res) => {
        if (!res.ok) throw new Error('فشل الطلب');
        return res.json();
      })
      .then((data) => setData(data))
      .catch((err) => setError(err.message))
      .finally(() => setLoading(false));
  }, [url]);

  return { data, loading, error };
}

export default useFetch;
5.2

استخدامه في عدة مكوّنات

JSX 
function UserProfile({ userId }) {
  const { data: user, loading, error } = useFetch(`https://api.example.com/users/${userId}`);
  if (loading) return <p>⏳ جارٍ التحميل...</p>;
  if (error) return <p>❌ {error}</p>;
  return <h2>{user.name}</h2>;
}

function PostsList() {
  const { data: posts, loading } = useFetch('https://api.example.com/posts');
  if (loading) return <p>⏳ جارٍ التحميل...</p>;
  return <ul>{posts.map((p) => <li key={p.id}>{p.title}</li>)}</ul>;
}
// نفس منطق التحميل والخطأ، بلا أي تكرار للكود بين المكوّنين!

الجزء السادس: useLocalStorage — Custom Hook للتخزين المحلي

6.1

بناء useLocalStorage

React 
// hooks/useLocalStorage.js
import { useState, useEffect } from 'react';

function useLocalStorage(key, initialValue) {
  const [value, setValue] = useState(() => {
    const saved = localStorage.getItem(key);
    return saved !== null ? JSON.parse(saved) : initialValue;
  });

  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(value));
  }, [key, value]);

  return [value, setValue];
}

export default useLocalStorage;
6.2

استخدامه لحفظ تفضيل المستخدم

JSX 
function App() {
  const [darkMode, setDarkMode] = useLocalStorage('darkMode', false);

  return (
    <div className={darkMode ? 'dark' : 'light'}>
      <button onClick={() => setDarkMode(!darkMode)}>
        {darkMode ? '☀️ وضع فاتح' : '🌙 وضع داكن'}
      </button>
    </div>
  );
}
// darkMode يُحفَظ تلقائياً في localStorage، ويبقى كما هو حتى بعد تحديث الصفحة بالكامل

الجزء السابع: المشروع — لوحة مقالات بـ Custom Hooks متعددة

نجمع كل ما تعلمناه: useFetch لجلب المقالات، useLocalStorage لحفظ المفضّلة، وuseRef لحقل البحث — في تطبيق واحد متكامل.

src/App.jsx

JSX 
import { useRef } from 'react';
import useFetch from './hooks/useFetch';
import useLocalStorage from './hooks/useLocalStorage';

function App() {
  const [favorites, setFavorites] = useLocalStorage('favorites', []);
  const { data: posts, loading, error } = useFetch(
    'https://jsonplaceholder.typicode.com/posts?_limit=10'
  );
  const searchRef = useRef(null);

  function toggleFavorite(id) {
    setFavorites((prev) =>
      prev.includes(id) ? prev.filter((f) => f !== id) : [...prev, id]
    );
  }

  if (loading) return <p>⏳ جارٍ تحميل المقالات...</p>;
  if (error) return <p>❌ {error}</p>;

  return (
    <div className="dashboard">
      <h1>📰 لوحة المقالات</h1>
      <input ref={searchRef} placeholder="ابحث..." />
      <p>⭐ المفضّلة: {favorites.length}</p>

      <ul>
        {posts.map((post) => (
          <li key={post.id}>
            <span>{post.title}</span>
            <button onClick={() => toggleFavorite(post.id)}>
              {favorites.includes(post.id) ? '⭐' : '☆'}
            </button>
          </li>
        ))}
      </ul>
    </div>
  );
}

export default App;

الجزء الثامن: تمارين الجلسة العشرين

✏️ تمرين 1 — useRef عملي15 دقيقة

  • مشغّل صوت/فيديو بسيط: زرا تشغيل وإيقاف يتحكّمان بعنصر <audio> أو <video> عبر ref
  • عدّاد نقرات لا يظهر في الواجهة — اطبعه في الكونسول فقط عندما يتجاوز 10 (بدون useState)

✏️ تمرين 2 — أول Custom Hook20 دقيقة

  • ابنِ useCounter Hook يُعيد [count, increment, decrement, reset]
  • ابنِ useWindowSize Hook يُعيد العرض والارتفاع الحاليين للنافذة ويتحدَّث مع resize
  • استخدم كليهما في مكوّن واحد للتأكد من عملهما

✏️ تمرين 3 — Custom Hooks متقدمة25 دقيقة

  • ابنِ useDebounce Hook يؤخِّر تحديث قيمة معينة بمقدار زمني محدَّد

الواجب: لوحة ملاحظات بـ Custom Hooks

أنشئ تطبيق React بـ Vite للملاحظات الشخصية، يجمع Custom Hooks متعددة:

◆ المتطلبات الوظيفية:

  • useFetch لجلب قائمة ملاحظات أولية من API تجريبي (JSONPlaceholder مثلاً)
  • useLocalStorage لحفظ أي ملاحظة يُضيفها المستخدم يدوياً بحيث تبقى بعد تحديث الصفحة
  • useRef للتركيز التلقائي على حقل إضافة الملاحظة عند تحميل الصفحة
  • Custom Hook إضافي من اختيارك (useToggle لوضع داكن، أو useCounter لعدد الملاحظات)

◆ متطلبات تقنية:

  • كل Custom Hook في ملف منفصل داخل src/hooks/
  • استخدام صحيح لمصفوفات الاعتماديات بلا تحذيرات في الكونسول
  • تحديث آمن للمصفوفات والكائنات (Immutability) في كل مكان
