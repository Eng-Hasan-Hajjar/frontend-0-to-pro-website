الجزء الأول: لماذا نكتب اختبارات؟

النوعيختبر ماذامثال
Unit Testدالة أو وحدة منفردة بمعزل عن الباقيدالة حسابية، دالة تنسيق تاريخ
Integration Testتفاعل عدة أجزاء معاًمكوّن يستدعي API ويعرض النتيجة
E2E (End-to-End)التطبيق كاملاً كما يراه مستخدم حقيقيتسجيل دخول ثم إتمام عملية شراء بالكامل

المشكلة بدون اختبارات

  • كل تعديل صغير في الكود يتطلّب فتح المتصفح والتحقّق يدوياً من كل شيء لا يزال يعمل
  • مع نمو المشروع، يصعب تذكّر كل الحالات التي يجب التحقّق منها
  • خطأ في مكوّن قد لا يظهر فوراً، بل بعد أسابيع، عند مستخدم حقيقي
1.2

الحل: اختبارات آلية

الاختبار الآلي كود يتحقّق من كود آخر — تُشغِّله بأمر واحد، ويُخبرك فوراً بالضبط أي جزء تعطَّل، دون فتح متصفح أو نقر يدوي واحد.

الجزء الثالث: أول اختبار وحدة (Unit Test)

3.1

اختبار دالة حسابية بسيطة

JavaScript 
// utils/math.js
function calculateTotal(price, quantity) {
  return price * quantity;
}

module.exports = { calculateTotal };
JavaScript 
// utils/math.test.js
const { calculateTotal } = require('./math');

test('يحسب السعر الإجمالي بشكل صحيح', () => {
  expect(calculateTotal(10, 3)).toBe(30);
});

test('يُعيد صفراً عند الكمية صفر', () => {
  expect(calculateTotal(10, 0)).toBe(0);
});
3.2

Matchers شائعة

Matcherالاستخدام
toBeمساواة تامة للقيم البدائية (أرقام، نصوص، منطقية)
toEqualمساواة "شكل" الكائنات والمصفوفات، لا نفس المرجع بالذاكرة
toContainالتحقّق أن مصفوفة أو نصاً يحتوي عنصراً معيّناً
toBeTruthy / toBeFalsyالتحقّق من قيمة صحيحة أو خاطئة منطقياً

الجزء الرابع: React Testing Library — اختبار المكوّنات

4.1

render وscreen

JSX 
import { render, screen } from '@testing-library/react';
import Welcome from './Welcome';

test('يعرض رسالة الترحيب', () => {
  render(<Welcome name="سارة" />);
  expect(screen.getByText('مرحباً سارة')).toBeInTheDocument();
});
💡 الفكرة

render يُصيِّر المكوّن في بيئة اختبار وهمية (لا متصفح حقيقي)، وscreen يمنحك أدوات للبحث عن محتوى داخل ما تمّ تصييره — تماماً كأنك "تنظر" للشاشة.

🖱️

الجزء الخامس: محاكاة تفاعل المستخدم

5.1

fireEvent للنقر

JSX 
import { render, screen, fireEvent } from '@testing-library/react';
import Counter from './Counter';

test('يزيد العدّاد عند الضغط على الزر', () => {
  render(<Counter />);

  const button = screen.getByText('زيادة +1');
  fireEvent.click(button);

  expect(screen.getByText('1')).toBeInTheDocument();
});
5.2

Mock Functions للتحقّق من الاستدعاء

JSX 
import { render, screen, fireEvent } from '@testing-library/react';
import ProductCard from './ProductCard';

test('يستدعي onAddToCart عند الضغط على الزر', () => {
  const handleAddToCart = jest.fn(); // دالة وهمية (Mock) لتتبّع الاستدعاء

  render(<ProductCard title="لابتوب" price={500} onAddToCart={handleAddToCart} />);

  fireEvent.click(screen.getByText('أضف للسلة'));

  expect(handleAddToCart).toHaveBeenCalledTimes(1);
});
💡 jest.fn()

يُنشئ دالة وهمية "تتذكّر" كل استدعاء لها: كم مرة استُدعيت، وبأي قيم بالضبط — مثالية للتحقّق من أن Props من نوع دالة (onClick، onSubmit...) استُدعيت بالشكل الصحيح.

🔗

الجزء السادس: اختبار مكوّن بـ Props كاملة

JSX 
import { render, screen } from '@testing-library/react';
import ProductCard from './ProductCard';

test('يعرض اسم المنتج وسعره الممرَّرين كـ Props', () => {
  render(<ProductCard title="لابتوب" price={500} onAddToCart={() => {}} />);

  expect(screen.getByText('لابتوب')).toBeInTheDocument();
  expect(screen.getByText('500 $')).toBeInTheDocument();
});

الجزء السابع: المشروع — اختبارات كاملة لمكوّن TaskItem

نختبر TaskItem من مشروع الجلسة 31 بثلاثة اختبارات تغطي العرض والتفاعل معاً.

TaskItem.test.jsx

JSX 
import { render, screen, fireEvent } from '@testing-library/react';
import TaskItem from './TaskItem';

const mockTask = { id: 1, title: 'تعلّم Jest', completed: false, priority: 'high' };

test('يعرض عنوان المهمة', () => {
  render(<TaskItem task={mockTask} onToggle={() => {}} onDelete={() => {}} />);
  expect(screen.getByText(/تعلّم Jest/)).toBeInTheDocument();
});

test('يستدعي onToggle بالمعرّف الصحيح عند الضغط على النص', () => {
  const handleToggle = jest.fn();
  render(<TaskItem task={mockTask} onToggle={handleToggle} onDelete={() => {}} />);

  fireEvent.click(screen.getByText(/تعلّم Jest/));
  expect(handleToggle).toHaveBeenCalledWith(mockTask.id);
});

test('يستدعي onDelete بالمعرّف الصحيح عند الضغط على زر الحذف', () => {
  const handleDelete = jest.fn();
  render(<TaskItem task={mockTask} onToggle={() => {}} onDelete={handleDelete} />);

  fireEvent.click(screen.getByText('🗑️'));
  expect(handleDelete).toHaveBeenCalledWith(mockTask.id);
});
✅ النتيجة

ثلاثة اختبارات صغيرة، كل واحد يتحقّق من سلوك واحد محدَّد بدقة — إن كسر تعديل مستقبلي أياً منها، ستعرف فوراً بالضبط أي جزء تعطَّل، بلا أي فتح للمتصفح.
