الجلسة الثالثة والعشرون — Nested Routes وLayouts في React Router

المسارات المتداخلة • Outlet • Layout Route • Index Route • Protected Routes

👤 المؤلف: Eng. Hasan Hajjar  ·  Software Engineer & AI Engineer

🎯 أهداف الجلسة الثالثة والعشرين

  • فهم مفهوم المسارات المتداخلة (Nested Routes) ومتى نحتاجها
  • استخدام Outlet لعرض المسارات الفرعية داخل تخطيط مشترك
  • بناء Layout Route بشريط تنقّل وتذييل ثابتين ومحتوى متغيّر
  • استخدام Index Route لتحديد الصفحة الافتراضية عند مسار أب
  • بناء آلية حماية مسارات (Protected Routes) لصفحات تتطلّب تسجيل دخول
  • دمج كل هذه المفاهيم في مشروع لوحة تحكّم (Dashboard) متكامل
🔁 مراجعة سريعة من الجلسة الثانية والعشرين

BrowserRouter يُغلِّف التطبيق بالكامل، وRoutes/Route تُعرِّف كل مسار ومكوّنه

Link وNavLink تنقّل فوري بلا إعادة تحميل، وNavLink يُنسِّق الرابط النشط تلقائياً

Route Parameters (:id) تسمح بصفحات ديناميكية، تُقرَأ عبر useParams

useNavigate يُتيح التنقّل البرمجي بعد أحداث (نموذج، تسجيل دخول...)

🎓 أهلاً بالتخطيطات المشتركة!

لاحظت أن كل صفحة في مشروعك السابق أعادت كتابة Navbar وFooter بنفسها؟ اليوم نحلّ هذا نهائياً.

Nested Routes تسمح بتعريف "تخطيط" واحد (Navbar + Footer + مساحة محتوى) يُشارَك بين عدة صفحات، بينما يتغيّر المحتوى الداخلي فقط حسب المسار.

🪆

الجزء الأول: ما هي Nested Routes ولماذا نحتاجها؟

1.1

المشكلة: تكرار التخطيط في كل صفحة

JSX 
// ❌ بدون Nested Routes: Navbar وFooter مكرَّران في كل صفحة!
function Home() {
  return (
    <>
      <Navbar />
      <h1>الرئيسية</h1>
      <Footer />
    </>
  );
}

function About() {
  return (
    <>
      <Navbar />
      <h1>من نحن</h1>
      <Footer />
    </>
  );
}
// أي تعديل على Navbar يتطلّب مراجعة كل صفحة على حدة!
1.2

الحل: مسار أب بتخطيط مشترك، ومسارات فرعية بداخله

React Router يسمح بتعريف Route "أب" يحتوي عنصر تخطيط (Layout)، وRoutes فرعية بداخله تُعرَض تلقائياً في المكان المخصَّص من هذا التخطيط.

🕳️

الجزء الثاني: Outlet

2.1

Outlet: أين تُعرض المسارات الفرعية

JSX 
// src/layouts/MainLayout.jsx
import { Outlet } from 'react-router-dom';
import Navbar from '../components/Navbar';
import Footer from '../components/Footer';

function MainLayout() {
  return (
    <>
      <Navbar />
      <main>
        <Outlet /> {/* هنا بالضبط يُعرض المسار الفرعي المطابق للرابط الحالي */}
      </main>
      <Footer />
    </>
  );
}

export default MainLayout;
2.2

ربط MainLayout بالمسارات الفرعية

JSX 
// src/App.jsx
import { Routes, Route } from 'react-router-dom';
import MainLayout from './layouts/MainLayout';
import About from './pages/About';

function App() {
  return (
    <Routes>
      <Route path="/" element={<MainLayout />}>
        <Route path="about" element={<About />} />
        {/* أي مسار هنا يُعرض تلقائياً داخل <Outlet /> في MainLayout */}
      </Route>
    </Routes>
  );
}
// الآن Navbar وFooter يُكتَبان مرة واحدة فقط في MainLayout!
🏠

الجزء الثالث: Index Route

3.1

المشكلة: أي صفحة تظهر عند "/" بالضبط؟

JSX 
// ❌ بدون index: زيارة "/" بالضبط لا تعرض أي محتوى داخل Outlet!
<Route path="/" element={<MainLayout />}>
  <Route path="about" element={<About />} />
</Route>
3.2

Index Route كحل

JSX 
// ✅ Index Route: المسار الافتراضي عند "/" بالضبط
<Route path="/" element={<MainLayout />}>
  <Route index element={<Home />} /> {/* index بدل path */}
  <Route path="about" element={<About />} />
</Route>
🔒

الجزء الرابع: حماية المسارات (Protected Routes)

4.1

مكوّن غلاف يتحقّق من تسجيل الدخول

JSX 
// src/components/ProtectedRoute.jsx
import { Navigate, Outlet } from 'react-router-dom';

function ProtectedRoute({ isLoggedIn }) {
  if (!isLoggedIn) {
    return <Navigate to="/login" replace />; // توجيه تلقائي، بلا إمكانية رجوع للصفحة المحمية
  }
  return <Outlet />; // مُسجَّل الدخول: اعرض المسار الفرعي المطلوب
}

export default ProtectedRoute;
4.2

استخدامه لحماية مجموعة مسارات

JSX 
function App() {
  const isLoggedIn = useAuth(); // Custom Hook افتراضي لحالة الدخول

  return (
    <Routes>
      <Route path="/" element={<MainLayout />}>
        <Route index element={<Home />} />
        <Route path="login" element={<Login />} />

        <Route element={<ProtectedRoute isLoggedIn={isLoggedIn} />}>
          <Route path="dashboard" element={<Dashboard />} />
          <Route path="settings" element={<Settings />} />
        </Route>
      </Route>
    </Routes>
  );
}
// dashboard وsettings محميّان: أي وصول بلا تسجيل دخول يُعاد توجيهه لـ /login تلقائياً
💻

الجزء الخامس: المشروع — لوحة تحكّم (Dashboard) متكاملة

نجمع كل المفاهيم: تخطيط رئيسي، تخطيط فرعي خاص بلوحة التحكّم، حماية مسارات، وIndex Route.

src/layouts/DashboardLayout.jsx

JSX 
import { Outlet, NavLink } from 'react-router-dom';

function DashboardLayout() {
  return (
    <div className="dashboard-layout">
      <aside className="sidebar">
        <NavLink to="/dashboard" end>نظرة عامة</NavLink>
        <NavLink to="/dashboard/orders">الطلبات</NavLink>
        <NavLink to="/dashboard/profile">الملف الشخصي</NavLink>
      </aside>
      <div className="content">
        <Outlet />
      </div>
    </div>
  );
}

export default DashboardLayout;
💡 خاصية end في NavLink

end تضمن أن هذا الرابط "نشط" فقط عند /dashboard بالضبط — لا عند /dashboard/orders أيضاً. بدونها، سيبقى رابط "نظرة عامة" مُنسَّقاً كنشط في كل صفحات اللوحة!

src/App.jsx — الربط الكامل (مسارات متداخلة داخل مسارات محمية)

JSX 
import { Routes, Route } from 'react-router-dom';
import MainLayout from './layouts/MainLayout';
import DashboardLayout from './layouts/DashboardLayout';
import ProtectedRoute from './components/ProtectedRoute';
import Home from './pages/Home';
import Login from './pages/Login';
import Overview from './pages/dashboard/Overview';
import Orders from './pages/dashboard/Orders';
import Profile from './pages/dashboard/Profile';

function App() {
  const isLoggedIn = true; // بيانات وهمية للتبسيط

  return (
    <Routes>
      <Route path="/" element={<MainLayout />}>
        <Route index element={<Home />} />
        <Route path="login" element={<Login />} />

        <Route element={<ProtectedRoute isLoggedIn={isLoggedIn} />}>
          <Route path="dashboard" element={<DashboardLayout />}>
            <Route index element={<Overview />} />
            <Route path="orders" element={<Orders />} />
            <Route path="profile" element={<Profile />} />
          </Route>
        </Route>
      </Route>
    </Routes>
  );
}

export default App;
🏛️ لاحظ التداخل الكامل

DashboardLayout نفسه مُتداخل داخل ProtectedRoute (حماية)، والذي بدوره متداخل داخل MainLayout (التخطيط العام) — ثلاث طبقات من Nested Routes تعمل معاً بسلاسة.

  • أنشئ قسم "المنتجات" بتخطيط فرعي خاص به (ProductsLayout بشريط تصنيفات جانبي)
  • مسارات فرعية: /products (index — كل المنتجات)، /products/electronics، /products/clothing

✏️ تمرين 3 — حماية مسار كاملة25 دقيقة

  • صفحة تسجيل دخول وهمية بـ checkbox "أنا مسجّل الدخول" يُحدِّث State
  • ProtectedRoute يحمي صفحة "الملف الشخصي" فعلياً بناءً على تلك الحالة
  • محاولة الوصول المباشر للصفحة المحمية بلا تسجيل دخول يجب أن تُعيد التوجيه لصفحة الدخول

الواجب: لوحة إدارة مهام محمية بالكامل

أنشئ تطبيق React بـ Vite للوحة إدارة مهام، بتخطيط Dashboard كامل ومحمي خلف تسجيل دخول:

◆ المتطلبات الوظيفية:

  • صفحة تسجيل دخول وهمية (بلا خادم فعلي — تحقّق بسيط في الواجهة كافٍ)
  • DashboardLayout بشريط جانبي (Sidebar) بمسارات فرعية: كل المهام، المهام المكتملة، الإحصائيات
  • كل مسارات اللوحة محمية بالكامل عبر ProtectedRoute
  • زر "تسجيل خروج" يُعيد التوجيه لصفحة تسجيل الدخول

◆ متطلبات تقنية:

  • استخدام Index Route لصفحة "نظرة عامة" الافتراضية داخل اللوحة
  • NavLink مع خاصية end حيث يلزم لتجنّب تنسيق خاطئ للرابط النشط
  • تنظيم الملفات: src/layouts/، src/pages/dashboard/، src/components/
