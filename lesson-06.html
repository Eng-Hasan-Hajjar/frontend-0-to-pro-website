1.3

viewport meta tag — الخطوة الأولى الإلزامية

هذا الوسم يُخبر المتصفح بأن يعرض الصفحة بعرض الجهاز الفعلي لا بعرض الكمبيوتر:

HTML 
<!-- يجب وضعه في <head> في كل صفحة HTML -->
<meta name="viewport" content="width=device-width, initial-scale=1.0" />

<!-- width=device-width: اجعل العرض = عرض الجهاز الفعلي -->
<!-- initial-scale=1.0: لا تُصغّر الصفحة عند فتحها -->

<!-- ❌ بدون هذا الوسم: المتصفح يُصغّر الصفحة لتبدو مثل الكمبيوتر -->
<!-- ✅ مع هذا الوسم: الصفحة تُعرض بحجم الجهاز الحقيقي -->
👁 المعاينة الحيّة
⚠️ تنبيه مهم

بدون viewport meta tag، Media Queries لن تعمل بشكل صحيح على الجوال.

الجزء الثاني: وحدات القياس المرنة

اختيار الوحدة الصحيحة هو أساس التصميم المتجاوب. كل وحدة لها دورها الصحيح:

2.1

وحدات مطلقة مقابل نسبية

النوعالوحدات وخصائصها
مطلقة (Absolute)px — ثابتة لا تتغير — مناسبة للحدود والظلال الدقيقة
نسبية (Relative)%, rem, em, vw, vh — تتغير بالسياق — مثالية للتجاوب
2.2

rem (Root Em) — الوحدة المُوصى بها للنصوص

rem = نسبة إلى حجم الخط الجذري (root) — وهو عادةً 16px في المتصفحات:

CSS 
/* حجم الخط الجذري = 16px (الافتراضي في المتصفح) */
html { font-size: 16px; }

/* حسابات rem */
/* 1rem  = 16px */
/* 1.5rem = 24px */
/* 2rem  = 32px */
/* 0.75rem = 12px */

/* خدعة 62.5% — تجعل 1rem = 10px للحساب الأسهل */
html { font-size: 62.5%; }  /* 16px × 62.5% = 10px */
body { font-size: 1.6rem; } /* 1.6 × 10 = 16px */
h1   { font-size: 3.6rem; } /* 3.6 × 10 = 36px */
h2   { font-size: 2.4rem; } /* 2.4 × 10 = 24px */

/* ميزة rem: عند تغيير html font-size تتغير كل القيم تلقائياً */
/* مثالي للمستخدمين الذين يغيرون حجم الخط في المتصفح */
2.3

em — نسبة إلى الأب

CSS 
/* em = نسبة إلى font-size الأب المباشر */
.parent {
  font-size: 20px;
}

.child {
  font-size: 1.5em;   /* 1.5 × 20px = 30px */
  padding: 1em;       /* 1 × 20px = 20px */
  margin: 0.5em;      /* 0.5 × 20px = 10px */
}

/* em مفيد للـ padding وmargin النسبية للخط */
.btn {
  font-size: 1rem;
  padding: 0.75em 1.5em; /* يتناسب مع حجم النص تلقائياً */
}

/* تحذير: em يتراكم في العناصر المتداخلة */
.outer { font-size: 1.2em; }   /* = 19.2px */
.inner { font-size: 1.2em; }   /* = 23px (تضاعف!) */
/* rem لا يتراكم — هذا هو سبب تفضيله */
2.4

vw و vh — نسبة إلى الشاشة

CSS 
/* vw = Viewport Width — 1vw = 1% من عرض الشاشة */
/* vh = Viewport Height — 1vh = 1% من ارتفاع الشاشة */

/* قسم Hero يملأ الشاشة كاملاً */
.hero {
  width: 100vw;
  height: 100vh;
}

/* عنوان يكبر وينكمش مع حجم الشاشة */
.responsive-title {
  font-size: 5vw;  /* 5% من عرض الشاشة */
  /* على 1200px: 60px | على 768px: 38px | على 375px: 19px */
}

/* ارتفاع كامل الصفحة ناقص الـ navbar */
.page-content {
  min-height: calc(100vh - 70px);
}

/* vmin = أصغر قيمة من vw وvh */
/* vmax = أكبر قيمة من vw وvh */
.square {
  width: 50vmin;   /* مربع يتناسب مع أصغر بُعد للشاشة */
  height: 50vmin;
}
2.5

النسبة المئوية % ودالة calc()

CSS 
/* % — نسبة إلى الأب */
.container {
  width: 90%;          /* 90% من عرض الأب */
  max-width: 1280px;   /* حد أقصى */
  margin: 0 auto;      /* توسيط أفقي */
}

/* calc() — حسابات مخلوطة */
.sidebar {
  width: calc(300px + 2rem);
}

.main {
  width: calc(100% - 300px - 30px); /* 100% - sidebar - gap */
}

.hero {
  height: calc(100vh - 70px); /* ارتفاع الشاشة - الـ navbar */
  padding: calc(2rem + 20px);
}

/* calc() تقبل: +  -  *  / */
/* يمكن مزج الوحدات: px + % + rem + vw */

2.6 — دوال CSS الحديثة: clamp() وmin() وmax()

هذه الدوال تُلغي الحاجة لكثير من استعلامات الوسائط وتجعل القيم تتقدر الأقلم تلقائياً:

CSS 
/* clamp(minimum, preferred, maximum) */
/* تضمن أن القيمة لا تقل عن minimum ولا تزيد عن maximum */

h1 {
  /* لا تقل عن 24px، تفضيلياً 5vw، لا تزيد عن 56px */
  font-size: clamp(24px, 5vw, 56px);
}

.container {
  /* عرض مرن بين 320px و1280px */
  width: clamp(320px, 90%, 1280px);
  margin: 0 auto;
}

.hero-text {
  font-size: clamp(1.5rem, 3vw + 1rem, 3rem);
  /* على الجوال: ~1.5rem | على الكمبيوتر: ~3rem */
}

/* min() — تأخذ الأصغر من القيمتين */
.box {
  width: min(500px, 90%); /* إما 500px أو 90% أيهما أصغر */
}

/* max() — تأخذ الأكبر من القيمتين */
.text {
  font-size: max(1rem, 2vw); /* لا تقل أبداً عن 1rem */
}
الوحدة / الدالةمتى تستخدمها؟
pxالحدود، الظلال، القيم الدقيقة الثابتة
remالخطوط والـ padding الرئيسية — الأكثر توصية
emالـ padding وmargin النسبية للعنصر نفسه
% العروض النسبية للحاويات
vw / vhقسم Hero الكامل، عناوين تتناسب مع الشاشة
frالأعمدة والصفوف في CSS Grid
calc()مزج وحدات مختلفة في حسابة واحدة
clamp()قيم مرنة بحدود دنيا وعليا — تُغني عن Media Queries
min() / max()تقييد الحد الأدنى أو الأقصى بمرونة

الجزء الثالث: Media Queries — استعلامات الوسائط

Media Queries هي آلية CSS التي تُطبّق أنماطاً مختلفة بناءً على خصائص الجهاز أو الشاشة.

3.1

بنية Media Query

CSS 
/* البنية الأساسية */
@media media-type and (condition) {
  /* CSS يُطبَّق فقط عند تحقق الشرط */
}

/* أمثلة */
@media screen and (max-width: 768px) {
  /* للشاشات بعرض أقل من أو يساوي 768px */
}

@media screen and (min-width: 1024px) {
  /* للشاشات بعرض أكبر من أو يساوي 1024px */
}

@media print {
  /* عند طباعة الصفحة — إخفاء الـ nav والإعلانات */
}

@media (orientation: landscape) {
  /* عند الوضع الأفقي للجهاز */
}
3.2

Mobile-First مقابل Desktop-First

Mobile-First هو المعيار الحديث: نكتب CSS الجوال أولاً ثم نُضيف تعديلات للشاشات الأكبر:

CSS 
/* ════════════════════════════════════════ */
/* ❌ Desktop-First (القديم) — لا يُنصح به */
/* ════════════════════════════════════════ */
.container {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr; /* كمبيوتر */
}

@media (max-width: 1024px) { /* تابلت */ }
@media (max-width: 768px)  { /* جوال كبير */ }
@media (max-width: 480px)  { /* جوال صغير */ }

/* ════════════════════════════════════════ */
/* ✅ Mobile-First (الحديث) — المُوصى به   */
/* ════════════════════════════════════════ */
.container {
  display: grid;
  grid-template-columns: 1fr; /* جوال — عمود واحد */
}

@media (min-width: 600px) {
  .container {
    grid-template-columns: 1fr 1fr; /* تابلت — عمودان */
  }
}

@media (min-width: 1024px) {
  .container {
    grid-template-columns: 1fr 1fr 1fr; /* كمبيوتر — 3 أعمدة */
  }
}
Mobile-First ✅Desktop-First ❌
يبدأ بـ min-widthيبدأ بـ max-width
يُحمّل أقل CSS على الجواليُحمّل CSS الكمبيوتر على الجوال أيضاً
أداء أفضل على الأجهزة الضعيفةأداء أسوأ
يُجبرك على التفكير بالجوال أولاًالجوال يأتي كفكرة لاحقة
معيار Google للـ SEOقديم ومهجور تدريجياً
3.3

النقاط الفاصلة القياسية (Breakpoints)

📐 نقاط التوقف القياسية — Mobile First
  /* الجوال أولاً — لا يحتاج Media Query */
  .element { /* 0px → ∞ */ }
  @media (min-width: 480px) { /* sm — جوال كبير */ }
  @media (min-width: 768px) { /* md — تابلت      */ }
  @media (min-width: 1024px){ /* lg — لابتوب     */ }
  @media (min-width: 1280px){ /* xl — كمبيوتر    */ }
  @media (min-width: 1536px){ /* 2xl — شاشة واسعة*/ }
  ──────────────────────────────────────────────
  0px    480px   768px   1024px  1280px  1536px
  |──────|───────|───────|───────|───────|────→
  📱جوال │ sm    │ md    │ lg    │  xl   │  2xl
CSS 
/* نظام نقاط التوقف الاحترافي — Mobile First */

/* Breakpoints كـ CSS Variables */
:root {
  --bp-sm: 480px;
  --bp-md: 768px;
  --bp-lg: 1024px;
  --bp-xl: 1280px;
}

/* الجوال (الافتراضي) */
.nav-links { display: none; }
.hamburger { display: block; }

/* تابلت: 768px+ */
@media (min-width: 768px) {
  .nav-links  { display: flex; }
  .hamburger  { display: none; }
}

/* كمبيوتر: 1024px+ */
@media (min-width: 1024px) {
  .container { max-width: 960px; }
}

/* شاشة واسعة: 1280px+ */
@media (min-width: 1280px) {
  .container { max-width: 1200px; }
}
3.4

Media Queries المتقدمة

CSS 
/* شرطان معاً — AND */
@media (min-width: 768px) and (max-width: 1024px) {
  /* للتابلت فقط بين 768px و1024px */
}

/* أحد الشرطين — OR (فاصلة) */
@media (max-width: 480px), (orientation: portrait) {
  /* للجوال الصغير أو الوضع العمودي */
}

/* نفي شرط — NOT */
@media not screen {
  /* لكل شيء ما عدا الشاشة */
}

/* كشف الـ hover (لا يدعمه الجوال) */
@media (hover: hover) {
  .btn:hover { background: #E8A020; }
  /* يُطبَّق فقط على أجهزة تدعم hover (ماوس) */
}

/* وضع الطباعة */
@media print {
  .navbar, .sidebar, .ads { display: none; }
  body { font-size: 12pt; color: black; }
  a { color: black; text-decoration: none; }
  a[href]::after { content: ' (' attr(href) ')'; }
}

/* تفضيل تقليل الحركة (Accessibility) */
@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}

/* الوضع الداكن */
@media (prefers-color-scheme: dark) {
  body { background: #1A1A2E; color: #CDD6F4; }
}

الصورة المتجاوبة الأساسية

4.2

object-fit للصور في حاويات ذات أبعاد ثابتة

CSS 
/* object-fit يتحكم في كيفية ملء الصورة للحاوية */
.card-image {
  width: 100%;
  height: 220px;          /* ارتفاع ثابت */
  object-fit: cover;      /* تغطي الحاوية دون تشوه — الأكثر استخداماً */
  object-fit: contain;    /* تظهر كاملة — قد تترك مساحة فارغة */
  object-fit: fill;       /* تملأ ولكن قد تتشوه */
  object-fit: none;       /* حجمها الطبيعي */
  object-position: center top; /* تحديد موضع الصورة */
}

/* أيقونة دائرية */
.avatar {
  width: 80px;
  height: 80px;
  border-radius: 50%;
  object-fit: cover;
}

الجزء الخامس: CSS Variables (المتغيرات المخصصة)

CSS Variables (أو Custom Properties) تُتيح تخزين قيم قابلة لإعادة الاستخدام في CSS — مثل المتغيرات في لغات البرمجة تماماً.

5.1

تعريف واستخدام المتغيرات

CSS 
/* تعريف المتغيرات في :root (النطاق العام) */
:root {
  /* ═══ نظام الألوان ═══ */
  --color-primary:    #1A3C5E;
  --color-secondary:  #2E75B6;
  --color-accent:     #E8A020;
  --color-bg:         #F0F6FF;
  --color-text:       #1A1A2E;
  --color-text-muted: #5A6A7A;
  --color-white:      #FFFFFF;

  /* ═══ الخطوط ═══ */
  --font-main:   "Cairo", sans-serif;
  --font-size-xs:   0.75rem;   /* 12px */
  --font-size-sm:   0.875rem;  /* 14px */
  --font-size-base: 1rem;      /* 16px */
  --font-size-lg:   1.125rem;  /* 18px */
  --font-size-xl:   1.25rem;   /* 20px */
  --font-size-2xl:  1.5rem;    /* 24px */
  --font-size-3xl:  1.875rem;  /* 30px */
  --font-size-4xl:  2.25rem;   /* 36px */

  /* ═══ المسافات ═══ */
  --space-1:  0.25rem;  /* 4px  */
  --space-2:  0.5rem;   /* 8px  */
  --space-3:  0.75rem;  /* 12px */
  --space-4:  1rem;     /* 16px */
  --space-5:  1.25rem;  /* 20px */
  --space-6:  1.5rem;   /* 24px */
  --space-8:  2rem;     /* 32px */
  --space-10: 2.5rem;   /* 40px */
  --space-12: 3rem;     /* 48px */

  /* ═══ الحدود والظلال ═══ */

  --radius-md:  10px;
  --radius-lg:  16px;

  --bp-md: 768px;
  --bp-lg: 1024px;
  --bp-xl: 1280px;

}

body {
  font-family: var(--font-main);
  background-color: var(--color-bg);
  color: var(--color-text);

✏️ تمرين 1 — وحدات القياس15 دقيقة

أنشئ صفحة اختبار الوحدات:

  • عنوان بـ font-size: clamp(1.5rem, 4vw, 3rem) — جرّب تغيير حجم النافذة
  • حاوية بـ width: clamp(300px, 90%, 1200px) تتكيف تلقائياً
  • زر بـ padding: 0.75em 1.5em — جرّب تغيير font-size وراقب الـ padding

✏️ تمرين 2 — Media Queries Mobile-First25 دقيقة

أنشئ صفحة Portfolio بثلاثة مراحل:

  • الجوال (افتراضي): بطاقات عمود واحد، nav مخفية، hamburger ظاهر
  • التابلت (768px+): بطاقات عمودان، nav أفقية، hamburger مخفي
  • الكمبيوتر (1024px+): بطاقات 3 أعمدة، تخطيط sidebar + main
  • اختبر بأدوات المطوّر في Chrome: F12 → رمز الجوال → اختر الأجهزة المختلفة

✏️ تمرين 3 — CSS Variables وDark Mode25 دقيقة

أنشئ نظام ألوان كاملاً بـ CSS Variables:

  • عرّف 12 متغيراً في :root: ألوان + خطوط + مسافات + ظلال
  • طبّق المتغيرات على جميع عناصر الصفحة (لا تستخدم قيماً مباشرة)
  • أضف Dark Mode بـ @media (prefers-color-scheme: dark)
  • أضف زر تبديل الوضع الداكن/الفاتح باستخدام JavaScript بسيط
  • تحقق: قيمة واحدة تتغير → تتغير الصفحة كلها تلقائياً
