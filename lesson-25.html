🎓 أهلاً بإدارة الحالة الاحترافية!

لاحظت أن عربة التسوّق أو أي حالة معقّدة تحتاج عدة دوال منفصلة بـ useState (إضافة، حذف، تعديل، تفريغ)؟

اليوم نتعلّم useReducer: طريقة تُجمِّع كل منطق التحديث في مكان واحد مركزي (Reducer)، بدل تبعثره — تماماً كالمنطق الذي رأيناه في مصفوفات setState(prev => ...) لكن بشكل أكثر تنظيماً وقابلية للاختبار.

🤹

الجزء الأول: متى تُصبح useState غير كافية؟

1.1

مثال: عربة تسوّق بعدة عمليات مترابطة

React 
function Cart() {
  const [items, setItems] = useState([]);

  function addItem(product) {
    setItems([...items, { ...product, quantity: 1 }]);
  }

  function removeItem(id) {
    setItems(items.filter((i) => i.id !== id));
  }

  function updateQuantity(id, quantity) {
    setItems(items.map((i) => (i.id === id ? { ...i, quantity } : i)));
  }

  function clearCart() {
    setItems([]);
  }

  // ... أربع دوال منفصلة، كل واحدة تُعيد بناء نفس المصفوفة بطريقتها الخاصة
}
1.2

المشكلة

  • منطق التحديث موزَّع في أربع دوال مختلفة داخل المكوّن نفسه
  • كل دالة "تعرف" شكل بيانات items وتتعامل معه بأسلوبها الخاص — تكرار وتشتّت
  • صعوبة اختبار هذا المنطق بمعزل عن الواجهة (كل دالة مربوطة بـ setState داخل المكوّن)
  • كلما زادت العمليات (خصم، عرض ترويجي، حد أقصى للكمية...) زاد التعقيد داخل المكوّن نفسه

الجزء الثاني: useReducer الأساسي

2.1

الأجزاء الثلاثة

JSX 
import { useReducer } from 'react';

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return state + 1;
    case 'decrement':
      return state - 1;
    case 'reset':
      return 0;
    default:
      return state;
  }
}

function Counter() {
  const [count, dispatch] = useReducer(reducer, 0);

  return (
    <div>
      <p>{count}</p>
      <button onClick={() => dispatch({ type: 'increment' })}>+</button>
      <button onClick={() => dispatch({ type: 'decrement' })}>−</button>
      <button onClick={() => dispatch({ type: 'reset' })}>إعادة تعيين</button>
    </div>
  );
}
💡 الأجزاء الثلاثة

count: الحالة الحالية (نتيجة reducer). dispatch: الطريقة الوحيدة لطلب تغيير — لا تُغيِّر الحالة مباشرة، بل "تُرسِل" أكشناً. reducer: دالة خالصة تُقرِّر شكل الحالة الجديدة بناءً على الحالة القديمة ونوع الأكشن.

📨

الجزء الثالث: الأكشنات (Actions)

3.1

نمط type + payload

كل أكشن هو كائن JavaScript عادي، بحقل type يُحدِّد "ماذا نريد أن يحدث"، وpayload اختياري يحمل البيانات اللازمة.

3.2

reducer بعدة أنواع أكشنات

JavaScript 
function cartReducer(state, action) {
  switch (action.type) {
    case 'ADD_ITEM':
      return [...state, { ...action.payload, quantity: 1 }];

    case 'REMOVE_ITEM':
      return state.filter((item) => item.id !== action.payload.id);

    case 'UPDATE_QUANTITY':
      return state.map((item) =>
        item.id === action.payload.id
          ? { ...item, quantity: action.payload.quantity }
          : item
      );

    case 'CLEAR_CART':
      return [];

    default:
      return state;
  }
}
⭐ لاحظ

كل منطق التحديث الآن مُجمَّع في دالة واحدة (cartReducer)، بدل أربع دوال متفرِّقة — أسهل للقراءة، وللاختبار بمعزل تام عن أي واجهة React.

الجزء الخامس: دمج useReducer مع useContext

دمج useReducer مع Context يمنحك نمطاً شبيهاً بمكتبات إدارة الحالة الكبيرة (مثل Redux)، لكن مدمج بالكامل في React نفسه بلا أي مكتبة خارجية.

5.1

CartContext بمنطق useReducer

JSX 
// context/CartContext.jsx
import { createContext, useContext, useReducer } from 'react';

const CartContext = createContext(null);

function cartReducer(state, action) {
  switch (action.type) {
    case 'ADD_ITEM':
      return [...state, { ...action.payload, quantity: 1 }];
    case 'REMOVE_ITEM':
      return state.filter((item) => item.id !== action.payload.id);
    case 'UPDATE_QUANTITY':
      return state.map((item) =>
        item.id === action.payload.id ? { ...item, quantity: action.payload.quantity } : item
      );
    case 'CLEAR_CART':
      return [];
    default:
      return state;
  }
}

export function CartProvider({ children }) {
  const [items, dispatch] = useReducer(cartReducer, []);
  return (
    <CartContext.Provider value={{ items, dispatch }}>
      {children}
    </CartContext.Provider>
  );
}

export function useCart() {
  const context = useContext(CartContext);
  if (!context) throw new Error('useCart يجب أن يُستخدَم داخل CartProvider');
  return context;
}
💡 القوة هنا

أي مكوّن في التطبيق يستطيع dispatch أكشنات لتعديل العربة أو قراءة items، بلا Prop Drilling وبلا تبعثر منطق — كل قواعد التحديث مُجمَّعة في cartReducer وحده.

الجزء السادس: المشروع — عربة تسوّق كاملة

تطبيق متجر بمكوّنين منفصلين (عرض المنتجات وملخّص العربة)، يتشاركان حالة واحدة مُدارة بالكامل عبر useReducer وContext.

src/App.jsx

JSX 
import { CartProvider } from './context/CartContext';
import ProductsGrid from './components/ProductsGrid';
import CartSummary from './components/CartSummary';

function App() {
  return (
    <CartProvider>
      <ProductsGrid />
      <CartSummary />
    </CartProvider>
  );
}

export default App;

src/components/ProductsGrid.jsx

JSX 
import { useCart } from '../context/CartContext';

const products = [
  { id: 1, name: 'لابتوب', price: 500 },
  { id: 2, name: 'هاتف ذكي', price: 300 },
  { id: 3, name: 'سماعات', price: 50 },
];

function ProductsGrid() {
  const { dispatch } = useCart();

  return (
    <div>
      <h2>🛍️ المنتجات</h2>
      {products.map((product) => (
        <div key={product.id} className="product-card">
          <span>{product.name} — {product.price} $</span>
          <button onClick={() => dispatch({ type: 'ADD_ITEM', payload: product })}>
            أضف للعربة
          </button>
        </div>
      ))}
    </div>
  );
}

export default ProductsGrid;

src/components/CartSummary.jsx

JSX 
import { useCart } from '../context/CartContext';

function CartSummary() {
  const { items, dispatch } = useCart();
  const total = items.reduce((sum, item) => sum + item.price * item.quantity, 0);

  return (
    <div className="cart-summary">
      <h2>🛒 العربة ({items.length})</h2>
      <ul>
        {items.map((item) => (
          <li key={item.id}>
            {item.name} × {item.quantity}
            <button
              onClick={() =>
                dispatch({
                  type: 'UPDATE_QUANTITY',
                  payload: { id: item.id, quantity: item.quantity + 1 },
                })
              }
            >
              +
            </button>
            <button onClick={() => dispatch({ type: 'REMOVE_ITEM', payload: { id: item.id } })}>
              🗑️
            </button>
          </li>
        ))}
      </ul>
      <p>الإجمالي: {total} $</p>
      <button onClick={() => dispatch({ type: 'CLEAR_CART' })}>تفريغ العربة</button>
    </div>
  );
}

export default CartSummary;

الجزء السابع: تمارين الجلسة الخامسة والعشرين

✏️ تمرين 1 — useReducer بسيط15 دقيقة

  • ابنِ reducer لإشارة مرور: أحمر ← أصفر ← أخضر ← أحمر...
  • أكشن واحد فقط بنوع "NEXT" ينقل للحالة التالية في الدورة

✏️ تمرين 2 — أكشنات متعددة20 دقيقة

  • ابنِ reducer لنموذج تسجيل بحقول متعددة (اسم، بريد، كلمة مرور)
  • ثلاثة أنواع أكشنات على الأقل: تحديث حقل، إعادة تعيين النموذج، تفعيل/تعطيل حالة التحميل

✏️ تمرين 3 — دمج مع Context25 دقيقة

  • خُذ أي Context بُني سابقاً بـ useState (مثل CartContext من جلسة سابقة إن وُجد)
  • حوِّله بالكامل ليستخدم useReducer بدلاً من useState، بلا تغيير في الواجهة الخارجية المُستخدَمة من useCart

الواجب: لوحة مهام Kanban مصغّرة

أنشئ تطبيق React بـ Vite للوحة مهام بنمط Kanban (أعمدة: قيد الانتظار، جارٍ العمل، مكتمل)، بمنطق useReducer مركزي كامل:

◆ المتطلبات الوظيفية:

  • إضافة مهمة جديدة لعمود "قيد الانتظار"
  • نقل مهمة بين الأعمدة الثلاثة (زر "التالي" ينقلها للعمود التالي)
  • حذف مهمة من أي عمود
  • عدّاد لعدد المهام في كل عمود

◆ متطلبات تقنية:

  • reducer واحد مركزي بأكشنات: ADD_TASK، MOVE_TASK، DELETE_TASK على الأقل
  • Context لمشاركة الحالة بين الأعمدة الثلاثة (كل عمود مكوّن منفصل)
  • استخدام Immutability الصحيح في كل حالة (بلا أي تعديل مباشر على المصفوفات)
