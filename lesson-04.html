الجزء الأول: لماذا Flexbox؟ المشكلة والحل

1.1

مشكلة التخطيط التقليدي

قبل Flexbox، كان المطوّرون يعانون كثيراً لإنجاز مهام بسيطة مثل:

المشكلةالحل القديم المُعقَّد
توسيط عنصر أفقياً وعمودياًmargin: auto + position absolute + transform (معقد جداً)
وضع عناصر بجانب بعضهاfloat + clearfix (هشّ وصعب التحكم)
توزيع المساحة بالتساويحساب النسب يدوياً بالبيكسل (% غير دقيق)
ترتيب العناصر دون تغيير HTMLمستحيل أو يتطلب JavaScript
تخطيط متجاوب (Responsive)تعليمات media query معقدة جداً
CSS 
/* ❌ الطريقة القديمة لتوسيط عنصر — مؤلمة */
.old-center {
  position: absolute;
  top: 50%;
  left: 50%;
  transform: translate(-50%, -50%);
}

/* ✅ الطريقة الحديثة مع Flexbox — سطران فقط! */
.flex-center {
  display: flex;
  justify-content: center;
  align-items: center;
}
1.2

ما هو Flexbox؟

Flexbox (Flexible Box Layout) هو نظام تخطيط أحادي البُعد في CSS — يُنظّم العناصر في صف أو عمود واحد مع التحكم الكامل في التوزيع والمحاذاة والمساحة.

💡 Flexbox أحادي البُعد — CSS Grid ثنائي الأبعاد

Flexbox: ممتاز لتخطيط صف أو عمود واحد — شريط التنقل، قائمة البطاقات، مجموعة الأزرار

CSS Grid: ممتاز لتخطيط الصفحة الكاملة بصفوف وأعمدة — سنتعلمه في الجلسة الخامسة

الفرق ليس تنافساً — كلاهما يُكمّل الآخر في المشروع الواحد

الجزء الثاني: المفاهيم الأساسية لـ Flexbox

2.1

الحاوية والعناصر (Container & Items)

Flexbox يعتمد على مستويين: الحاوية الأم التي تتحكم في التخطيط، والعناصر الأبناء التي تُرتَّب بداخلها:

HTML 
<!-- HTML -->
<div class="container">   <!-- الحاوية (Flex Container) -->
  <div class="item">1</div>  <!-- عنصر (Flex Item) -->
  <div class="item">2</div>  <!-- عنصر (Flex Item) -->
  <div class="item">3</div>  <!-- عنصر (Flex Item) -->
</div>
CSS 
/* CSS */
.container {
  display: flex;  /* هذا السطر الوحيد يُفعّل Flexbox */
  /* الآن جميع الأبناء المباشرين أصبحوا Flex Items */
}

/* ملاحظة: فقط الأبناء المباشرون يتأثرون */
/* الأحفاد (أبناء الأبناء) لا يتأثرون */
👁 المعاينة الحيّة
2.2

المحاور (Axes)

Flexbox يعمل على محورين — فهمهما هو مفتاح فهم كل خصائص Flexbox:

📐 محاور Flexbox
  flex-direction: row (الافتراضي)
  ┌─────────────────────────────────────┐
  │  Main Axis (المحور الرئيسي) ───────→ │
  │  ┌─────┐  ┌─────┐  ┌─────┐         │
  │  │  1  │  │  2  │  │  3  │   ↕     │
  │  └─────┘  └─────┘  └─────┘         │
  │                          Cross Axis  │
  └─────────────────────────────────────┘
  flex-direction: column
  ┌────────────────────┐
  │  ┌──────────────┐  │  ↕ Main Axis
  │  │      1       │  │
  │  └──────────────┘  │
  │  ┌──────────────┐  │
  │  │      2       │  │
  │  └──────────────┘  │
  │  Cross Axis ──────→ │
  └────────────────────┘
المصطلحالمعنى
Main Axis (المحور الرئيسي)اتجاه تدفق العناصر — يتحدد بـ flex-direction
Cross Axis (المحور العرضي)المحور العمودي على المحور الرئيسي
Main Start / Main Endبداية ونهاية المحور الرئيسي
Cross Start / Cross Endبداية ونهاية المحور العرضي
justify-contentتوزيع العناصر على المحور الرئيسي
align-itemsمحاذاة العناصر على المحور العرضي
🏠

الجزء الثالث: خصائص الحاوية (Container Properties)

هذه الخصائص تُكتب على الحاوية الأم (العنصر الذي لديه display: flex).

3.1

flex-direction (اتجاه المحور الرئيسي)

CSS 
.container {
  display: flex;

  flex-direction: row;            /* ← الافتراضي: عناصر بجانب بعضها من اليسار لليمين */
  flex-direction: row-reverse;    /* ← عناصر بجانب بعضها من اليمين لليسار */
  flex-direction: column;         /* ← عناصر فوق بعضها من الأعلى للأسفل */
  flex-direction: column-reverse; /* ← عناصر فوق بعضها من الأسفل للأعلى */
}
القيمةالنتيجة البصرية
row[ 1 ][ 2 ][ 3 ] ← الأكثر استخداماً لشريط التنقل
row-reverse[ 3 ][ 2 ][ 1 ] ← مفيد للغات RTL أحياناً
column[ 1 ] فوق [ 2 ] فوق [ 3 ] ← للقوائم العمودية
column-reverse[ 3 ] فوق [ 2 ] فوق [ 1 ] ← نادر الاستخدام
3.2

justify-content (توزيع على المحور الرئيسي)

تتحكم في كيفية توزيع العناصر على المحور الرئيسي (أفقياً في row، عمودياً في column):

CSS 
.container {
  display: flex;

  justify-content: flex-start;    /* |[1][2][3]          | الافتراضي — بداية المحور */
  justify-content: flex-end;      /* |          [1][2][3]| نهاية المحور */
  justify-content: center;        /* |    [1][2][3]      | توسيط */
  justify-content: space-between; /* |[1]    [2]    [3]  | مسافات بين العناصر */
  justify-content: space-around;  /* | [1]  [2]  [3]     | مسافات حول العناصر */
  justify-content: space-evenly;  /* |  [1]  [2]  [3]    | مسافات متساوية تماماً */
}
🔀 justify-content — التوزيع على المحور الرئيسي
flex-start:    │█ █ █ ·  ·  ·  ·  ·│
flex-end:      │·  ·  ·  ·  · █ █ █│
center:        │·  ·   █ █ █   ·  ·│
space-between: │█ ·  ·  · █ ·  ·  · █│
space-around:  │· █ ·  ·  █ ·  ·  █ ·│
space-evenly:  │·  █  ·  █  ·  █  ·│
3.3

align-items (محاذاة على المحور العرضي)

تتحكم في محاذاة العناصر على المحور العرضي (عمودياً في row، أفقياً في column):

CSS 
.container {
  display: flex;
  height: 200px; /* مهم لرؤية تأثير align-items */

  align-items: stretch;     /* ← الافتراضي: تمتد لملء ارتفاع الحاوية */
  align-items: flex-start;  /* ← تلتصق بأعلى الحاوية */
  align-items: flex-end;    /* ← تلتصق بأسفل الحاوية */
  align-items: center;      /* ← توسط عمودي */
  align-items: baseline;    /* ← محاذاة خطوط قاعدة النص */
}
↕️ align-items — المحاذاة على المحور العرضي
← الأعمدة تملأ كامل الارتفاع
stretch:    ┌────┬────┬────┐  
            │    │    │    │
            └────┴────┴────┘
flex-start: ┌────┬────┬────┐
            │ 1  │ 2  │ 3  │
            │    │    │    │
flex-end:   │    │    │    │
            │ 1  │ 2  │ 3  │
            └────┴────┴────┘

← في المنتصف

center:     │    │    │    │
            │ 1  │ 2  │ 3  │  
            │    │    │    │
3.4

flex-wrap (التفاف العناصر)

CSS 
.container {
  display: flex;

  flex-wrap: nowrap;       /* الافتراضي — كل العناصر في سطر واحد (قد تضغط) */
  flex-wrap: wrap;         /* العناصر تنتقل لسطر جديد عند الحاجة ✅ */
  flex-wrap: wrap-reverse; /* تلتف لكن من الأسفل للأعلى */
}

/* مثال شبكة بطاقات متجاوبة */
.cards-container {
  display: flex;
  flex-wrap: wrap;
  gap: 20px;
}

.card {
  width: 300px; /* كل بطاقة 300px — تلتف تلقائياً عند الحاجة */
}
3.5

gap (المسافة بين العناصر)

CSS 
.container {
  display: flex;

  gap: 20px;           /* مسافة 20px بين جميع العناصر (صفوفاً وأعمدةً) */
  gap: 10px 20px;      /* 10px بين الصفوف | 20px بين الأعمدة */
  row-gap: 10px;       /* مسافة بين الصفوف فقط */
  column-gap: 20px;    /* مسافة بين الأعمدة فقط */
}

/* gap أفضل من margin للمسافات داخل Flexbox */
/* لأنه لا يُضيف مسافة خارج الحاوية */
3.6

align-content (توزيع الصفوف المتعددة)

تعمل فقط عند وجود flex-wrap: wrap وأكثر من صف واحد:

CSS 
.container {
  display: flex;
  flex-wrap: wrap;
  height: 400px;

  align-content: flex-start;    /* الصفوف تلتصق بالأعلى */
  align-content: flex-end;      /* الصفوف تلتصق بالأسفل */
  align-content: center;        /* الصفوف في المنتصف */
  align-content: space-between; /* الصفوف موزعة */
  align-content: space-around;  /* مسافات حول الصفوف */
  align-content: stretch;       /* الافتراضي: الصفوف تمتد */
}
📌 ملخص سريع: justify-content مقابل align-content مقابل align-items

justify-content: توزيع العناصر على المحور الرئيسي (دائماً يعمل)

align-items: محاذاة عناصر صف واحد على المحور العرضي (دائماً يعمل)

align-content: توزيع الصفوف المتعددة على المحور العرضي (يعمل فقط مع flex-wrap: wrap)

🎛️

الجزء الرابع: خصائص العناصر (Item Properties)

هذه الخصائص تُكتب على عناصر Flex Items الأبناء — لا على الحاوية.

4.1

flex-grow (النمو)

يحدد كم يأخذ العنصر من المساحة الفارغة المتبقية — القيمة الافتراضية: 0 (لا ينمو):

CSS 
.container { display: flex; }

/* مثال 1: عنصر واحد ينمو */
.item-a { flex-grow: 0; }  /* لا ينمو — يبقى بحجمه الطبيعي */
.item-b { flex-grow: 1; }  /* يأخذ كل المساحة الفارغة */
.item-c { flex-grow: 0; }  /* لا ينمو */

/* مثال 2: توزيع نسبي */
.sidebar  { flex-grow: 1; }  /* يأخذ ثلث المساحة */
.main     { flex-grow: 2; }  /* يأخذ ثلثي المساحة */

/* مثال 3: توزيع متساوٍ لجميع العناصر */
.nav-item { flex-grow: 1; }  /* جميع عناصر القائمة بحجم متساوٍ */
📏 flex-grow — توزيع المساحة الفارغة
flex-grow: 0 (الافتراضي)
┌──────┬──────┬──────┬─────────────────┐
│  A   │  B   │  C   │  مساحة فارغة    │
└──────┴──────┴──────┴─────────────────┘
A:0, B:1, C:0
┌──────┬───────────────────────┬──────┐
│  A   │           B           │  C   │
└──────┴───────────────────────┴──────┘
A:1, B:2, C:1  (النسب: 1:2:1)
┌────────┬──────────────────┬────────┐
│   A    │        B         │   A    │
└────────┴──────────────────┴────────┘
4.2

flex-shrink (الانكماش)

CSS 
/* flex-shrink يحدد كم ينكمش العنصر عند ضيق المساحة */
/* القيمة الافتراضية: 1 (ينكمش بالتساوي مع الآخرين) */

.item-important {
  flex-shrink: 0;  /* لا ينكمش أبداً — يحافظ على حجمه */
}

.item-flexible {
  flex-shrink: 2;  /* ينكمش بمعدل ضعف الآخرين */
}

/* استخدام شائع: الشعار في الشريط لا ينكمش */
.logo {
  flex-shrink: 0;
  width: 150px;
}
4.3

flex-basis (الحجم الأساسي)

CSS 
/* flex-basis يحدد الحجم الابتدائي للعنصر قبل توزيع المساحة */
/* يشبه width في row، وheight في column */

.item {
  flex-basis: auto;    /* الافتراضي — يعتمد على المحتوى */
  flex-basis: 200px;   /* حجم ثابت كنقطة انطلاق */
  flex-basis: 30%;     /* نسبة مئوية من الحاوية */
  flex-basis: 0;       /* يبدأ من الصفر — flex-grow يتحكم بالكامل */
}
4.4

الخاصية المختصرة: flex

flex هي الاختصار لـ flex-grow + flex-shrink + flex-basis معاً:

CSS 
/* flex: grow shrink basis */
.item {
  flex: 1;          /* = flex: 1 1 0%  — ينمو وينكمش بحرية */
  flex: 2;          /* = flex: 2 1 0%  — ينمو بضعف الآخرين */
  flex: auto;       /* = flex: 1 1 auto */
  flex: none;       /* = flex: 0 0 auto — لا ينمو ولا ينكمش */
  flex: 0 0 200px;  /* حجم ثابت 200px لا يتغير أبداً */
}

/* المثال الأكثر شيوعاً: توزيع متساوٍ */
.nav-item {
  flex: 1;  /* جميع العناصر تأخذ مساحة متساوية */
}

/* تخطيط sidebar + main content */
.sidebar { flex: 0 0 280px; }  /* sidebar ثابت 280px */
.main    { flex: 1; }          /* main يأخذ الباقي */
4.5

align-self (محاذاة عنصر واحد)

يتجاوز align-items للحاوية ويُطبَّق على عنصر واحد فقط:

CSS 
.container {
  display: flex;
  align-items: center;  /* جميع العناصر في المنتصف */
  height: 200px;
}

.item-special {
  align-self: flex-start;  /* هذا العنصر فقط في الأعلى */
  /* align-self: flex-end | center | stretch | baseline | auto */
}

/* مثال: زر في أسفل بطاقة */
.card {
  display: flex;
  flex-direction: column;
}
.card-button {
  align-self: flex-end;  /* الزر في يسار البطاقة */
  margin-top: auto;      /* يدفع الزر لأسفل البطاقة */
}
4.6

order (ترتيب العرض)

CSS 
/* order يغيّر ترتيب ظهور العنصر دون تغيير HTML */
/* القيمة الافتراضية: 0 */
/* العناصر مرتبة من أصغر قيمة order إلى أكبرها */

.item-a { order: 2; }   /* يظهر ثالثاً */
.item-b { order: 0; }   /* يظهر أولاً (الافتراضي) */
.item-c { order: 1; }   /* يظهر ثانياً */
.item-d { order: -1; }  /* يظهر قبل الجميع (قيمة سالبة) */

/* النتيجة: item-d, item-b, item-c, item-a */

/* استخدام شائع: تغيير الترتيب في الجوال */
@media (max-width: 768px) {
  .hero-image { order: -1; } /* الصورة تأتي أولاً في الجوال */
  .hero-text  { order: 1; }  /* النص يأتي ثانياً */
}

الجزء السادس: أنماط Flexbox الشائعة

هذه هي الوصفات الجاهزة التي ستستخدمها مراراً في مشاريعك الحقيقية:

النمط 1 — التوسيط الكامل (أفقياً وعمودياً)

CSS 
/* ✅ أسهل طريقة لتوسيط أي شيء في CSS */
.centered-container {
  display: flex;
  justify-content: center;  /* توسيط أفقي */
  align-items: center;      /* توسيط عمودي */
  min-height: 100vh;        /* ارتفاع كامل الشاشة */
}

/* مثال: شاشة تسجيل الدخول */
.login-page {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 100vh;
  background: linear-gradient(135deg, #1A3C5E, #2E75B6);
}

.login-card {
  background: white;
  padding: 40px;
  border-radius: 16px;
  width: 400px;
  box-shadow: 0 20px 60px rgba(0,0,0,0.2);
}

النمط 2 — شريط التنقل (Navbar)

CSS 
/* شريط تنقل: الشعار يساراً، الروابط يميناً */
.navbar {
  display: flex;
  justify-content: space-between;  /* شعار + روابط على الطرفين */
  align-items: center;             /* محاذاة عمودية في المنتصف */
  padding: 16px 40px;
  background-color: #1A3C5E;
  position: sticky;
  top: 0;
  z-index: 100;
}

.navbar-logo {
  color: white;
  font-size: 24px;
  font-weight: 700;
  flex-shrink: 0;  /* الشعار لا ينكمش */

الجزء الثامن: تمارين الجلسة الرابعة

✏️ تمرين 1 — التوسيط والمحاذاة15 دقيقة

أنشئ صفحة فيها:

  • مربع 200×200px في مركز الشاشة تماماً (أفقياً وعمودياً)
  • 5 أزرار موزّعة بالتساوي في شريط أفقي (space-evenly)
  • 3 بطاقات: الأولى في الأعلى، الثانية في المنتصف، الثالثة في الأسفل (align-self)
  • قائمة أفقية من 4 روابط مع justify-content: space-between

✏️ تمرين 2 — flex-grow وflex-shrink20 دقيقة

أنشئ تخطيط صفحة مقالة يحتوي على:

  • Sidebar ثابت بعرض 250px (flex: 0 0 250px) لا ينكمش أبداً
  • Main content يأخذ كل المساحة المتبقية (flex: 1)
  • داخل Main: عنوان + 3 فقرات + قسم تعليقات
  • داخل Sidebar: قائمة بـ 5 روابط + بطاقة إعلانية صغيرة
  • استخدم gap: 30px بين الـ sidebar والـ main

✏️ تمرين 3 — شبكة بطاقات متجاوبة25 دقيقة

أنشئ قسم «فريق العمل» يحتوي على:

  • 6 بطاقات لأعضاء الفريق (اسم + مسمى + صورة placeholder)
  • البطاقات تلتف تلقائياً: 3 في الصف الكبير، 2 في المتوسط، 1 في الصغير
  • كل بطاقة: flex-direction column + توسيط align-items center
  • hover effect: ارتفاع خفيف مع تغيير ظل (transition: 0.3s)
  • order: اجعل مدير الفريق يظهر أولاً بغض النظر عن ترتيبه في HTML

المحاورMain Axis (اتجاه flex-direction) + Cross Axis (العمودي عليه)
flex-directionrow | row-reverse | column | column-reverse
justify-contentflex-start | flex-end | center | space-between | space-around | space-evenly
align-itemsstretch | flex-start | flex-end | center | baseline
flex-wrapnowrap | wrap — لالتفاف العناصر عند الضيق
gapمسافة بين العناصر — أفضل من margin للـ Flex
align-contentتوزيع الصفوف المتعددة (مع flex-wrap فقط)
flex-growكم ينمو العنصر لملء المساحة الفارغة
flex-shrinkكم ينكمش عند الضيق (1 افتراضي)
flex-basisالحجم الابتدائي قبل النمو/الانكماش
flexاختصار: grow shrink basis — flex: 1 الأكثر شيوعاً
align-selfمحاذاة عنصر واحد تتجاوز align-items
orderترتيب العرض دون تغيير HTML
