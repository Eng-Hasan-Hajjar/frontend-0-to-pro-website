🎓 أهلاً بمرحلة TypeScript!

كل الكود الذي كتبناه حتى الآن كان JavaScript خالصاً — مرناً وسريع الكتابة، لكنه لا يُخبرك بأخطائك إلا وقت التشغيل، أحياناً بعد أن يصل الخطأ لمستخدم حقيقي!

TypeScript يُضيف "نظام أنواع" فوق JavaScript، يكتشف فئة كاملة من الأخطاء وأنت تكتب الكود، قبل أي تشغيل إطلاقاً. اليوم نأخذ أولى خطواتنا فيه.

🤔

الجزء الأول: لماذا TypeScript؟

1.1

المشكلة: أخطاء لا تظهر إلا وقت التشغيل

JavaScript 
// JavaScript عادي — هذا الخطأ لن يظهر إلا وقت التشغيل!
function calculateTotal(price, quantity) {
  return price * quantity;
}

calculateTotal(10, "خمسة"); // NaN — لا يوجد أي تحذير حتى وقت التشغيل!
1.2

الحل: نظام أنواع يكتشف الخطأ فوراً

TypeScript 
// TypeScript — الخطأ يظهر فوراً أثناء الكتابة، قبل أي تشغيل!
function calculateTotal(price: number, quantity: number): number {
  return price * quantity;
}

calculateTotal(10, "خمسة");
// ❌ خطأ فوري في المحرّر:
// Argument of type 'string' is not assignable to parameter of type 'number'
💡 الفكرة الجوهرية

TypeScript لا "يُشغِّل" الكود ليكتشف الخطأ — يكتشفه فقط بقراءة الأنواع، أثناء الكتابة نفسها، غالباً مباشرة داخل المحرّر (VS Code) قبل حفظ الملف حتى.

الجزء الثاني: الإعداد والتحويل (Compilation)

2.1

تثبيت TypeScript

Shell 
npm install -g typescript
tsc --version
2.2

من .ts إلى .js

Shell 
tsc app.ts
# ينتج app.js — كود JavaScript خالص قابل للتشغيل في أي متصفح أو Node.js
💡 مهم

المتصفحات لا "تفهم" TypeScript مباشرة أبداً — المُحوِّل (tsc) يُترجمه لـ JavaScript عادي أولاً، وهذا الناتج (app.js) هو ما يعمل فعلياً. في مشاريع Vite أو Next.js، هذا التحويل يحدث تلقائياً في الخلفية.

3.1

let name: string = 'سارة';

3.2

any مقابل unknown

TypeScript 
let value: any = 'نص'; // ⚠️ any يُعطّل نظام الأنواع بالكامل — تجنّبه قدر الإمكان
value = 42; // لا خطأ! any يقبل أي شيء، وهذا يُفقدنا كل فائدة TypeScript

let safeValue: unknown = 'نص'; // ✅ unknown أكثر أماناً
// safeValue.toUpperCase(); // ❌ خطأ: يجب التحقّق من النوع أولاً

if (typeof safeValue === 'string') {
  safeValue.toUpperCase(); // ✅ الآن مسموح، بعد التأكد من النوع فعلياً
}
⭐ القاعدة الذهبية

تجنّب any قدر الإمكان — استخدامه يُعيدك عملياً لـ JavaScript الخالص بلا أي حماية. عندما لا تعرف النوع مسبقاً، unknown أكثر أماناً لأنه يُجبرك على التحقّق قبل الاستخدام.

الجزء الرابع: Interfaces وType Aliases

4.1

وصف شكل كائن بـ interface

TypeScript 
interface User {
  name: string;
  age: number;
  email?: string; // ? يعني: حقل اختياري
}

function greetUser(user: User): string {
  return `مرحباً ${user.name}!`;
}

greetUser({ name: 'أحمد', age: 30 }); // ✅ صحيح، email اختياري فلا حاجة له
greetUser({ name: 'أحمد' }); // ❌ خطأ: الحقل age مطلوب ومفقود!
4.2

type مقابل interface

TypeScript 
type UserType = {
  name: string;
  age: number;
};

interface UserInterface {
  name: string;
  age: number;
}
// الفرق الرئيسي: interface يمكن "توسيعها" (extends) بسهولة أكبر، وتُفضَّل لوصف الكائنات
// type أكثر مرونة لأنواع أخرى (اتحادات Union، تقاطعات Intersection) سنراها لاحقاً

الجزء السادس: TypeScript مع React

6.1

كتابة Props بـ interface

JSX 
interface CardProps {
  title: string;
  price: number;
  onAddToCart: (id: number) => void;
}

function ProductCard({ title, price, onAddToCart }: CardProps) {
  return (
    <div className="card">
      <h3>{title}</h3>
      <p>{price} $</p>
      <button onClick={() => onAddToCart(1)}>أضف للسلة</button>
    </div>
  );
}
6.2

useState مع أنواع محدَّدة

JSX 
import { useState } from 'react';

function Counter() {
  const [count, setCount] = useState<number>(0); // النوع محدَّد صراحة

  return <button onClick={() => setCount(count + 1)}>{count}</button>;
}
💡 Type Inference

TypeScript غالباً يستنتج النوع تلقائياً من القيمة الابتدائية — useState(0) يكفي عادة ليُستنتَج النوع number بلا كتابته صراحة. التصريح الصريح <number> مفيد للوضوح، أو عند الحاجة لأنواع أكثر تعقيداً.

الجزء السابع: المشروع — تحويل مكوّن من JavaScript إلى TypeScript

نأخذ مكوّناً حقيقياً بـ JavaScript ونحوّله بالكامل — Props والبيانات والدوال — إلى TypeScript كامل الأنواع.

قبل: ProductCard.jsx

JSX 
function ProductCard({ title, price, onAddToCart }) {
  return (
    <div className="card">
      <h3>{title}</h3>
      <p>{price} $</p>
      <button onClick={() => onAddToCart(title)}>أضف للسلة</button>
    </div>
  );
}

export default ProductCard;

بعد: ProductCard.tsx

JSX 
interface ProductCardProps {
  title: string;
  price: number;
  onAddToCart: (title: string) => void;
}

function ProductCard({ title, price, onAddToCart }: ProductCardProps) {
  return (
    <div className="card">
      <h3>{title}</h3>
      <p>{price} $</p>
      <button onClick={() => onAddToCart(title)}>أضف للسلة</button>
    </div>
  );
}

export default ProductCard;

App.tsx — الاستخدام ببيانات مكتوبة بأنواع

JSX 
interface Product {
  id: number;
  title: string;
  price: number;
}

const products: Product[] = [
  { id: 1, title: 'لابتوب', price: 500 },
  { id: 2, title: 'هاتف ذكي', price: 300 },
];

function App() {
  function handleAddToCart(title: string) {
    console.log(`أُضيف للسلة: ${title}`);
  }

  return (
    <div>
      {products.map((product) => (
        <ProductCard
          key={product.id}
          title={product.title}
          price={product.price}
          onAddToCart={handleAddToCart}
        />
      ))}
    </div>
  );
}

export default App;
