🎓 أهلاً بتطبيقات متعددة الصفحات!

كل تطبيقاتنا حتى الآن كانت "صفحة واحدة" فعلياً — واجهة واحدة تتغيّر محتوياتها.

لكن أي موقع حقيقي يحتاج عناوين URL مختلفة لكل صفحة (/about، /products/42...). اليوم نتعلّم React Router: المكتبة الرسمية شبه القياسية للتنقّل في تطبيقات React.

🧭

الجزء الأول: لماذا نحتاج Router؟

1.1

ما هو Single Page Application (SPA)؟

تطبيقات React عادةً هي "صفحة واحدة" حقيقةً — ملف HTML واحد فقط يُحمَّل مرة، وJavaScript يتولّى تبديل المحتوى المعروض دون أي إعادة تحميل من الخادم.

1.2

المشكلة بدون Router

  • وسم <a href="..."> العادي يُعيد تحميل الصفحة بالكامل من الخادم — تفقد كل حالة React (State، Context، كل شيء!)
  • بلا Router، لا يوجد عنوان URL مختلف لكل "صفحة" — لا يمكن مشاركة رابط مباشر لصفحة منتج معيّن مثلاً
  • زر "رجوع" في المتصفح لن يعمل بشكل متوقَّع
💡 الحل

React Router يُراقب عنوان URL في المتصفح، ويعرض المكوّن المناسب له — كل ذلك عبر JavaScript فقط، بلا أي طلب جديد للخادم.

📦

الجزء الثاني: التثبيت والإعداد

2.1

تثبيت react-router-dom

Shell 
npm install react-router-dom
2.2

BrowserRouter: تغليف التطبيق بالكامل

JSX 
// src/main.jsx
import { BrowserRouter } from 'react-router-dom';
import ReactDOM from 'react-dom/client';
import App from './App';

ReactDOM.createRoot(document.getElementById('root')).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
);
💡 لماذا في main.jsx؟

BrowserRouter يجب أن يُحيط التطبيق بالكامل من الأعلى، تماماً مثل أي Context Provider، حتى تتمكّن كل المكوّنات الداخلية من استخدام أدوات التنقّل.

الجزء الثالث: Routes وRoute

3.1

تعريف المسارات الأساسية

JSX 
// src/App.jsx
import { Routes, Route } from 'react-router-dom';
import Home from './pages/Home';
import About from './pages/About';
import Contact from './pages/Contact';

function App() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/about" element={<About />} />
      <Route path="/contact" element={<Contact />} />
    </Routes>
  );
}

export default App;
💡 كيف تعمل

Routes تفحص عنوان URL الحالي، وتعرض أول Route يُطابق path — element هو المكوّن الذي يظهر لهذا المسار بالذات.

3.2

صفحة 404 (Not Found)

JSX 
import NotFound from './pages/NotFound';

function App() {
  return (
    <Routes>
      <Route path="/" element={<Home />} />
      <Route path="/about" element={<About />} />
      <Route path="*" element={<NotFound />} /> {/* أي مسار غير معرَّف أعلاه */}
    </Routes>
  );
}

الجزء الرابع: Link وNavLink

4.1

لماذا لا نستخدم وسم <a> عادي

HTML 
<!-- ❌ يُعيد تحميل الصفحة بالكامل من الخادم — تفقد كل حالة React! -->
<a href="/about">من نحن</a>
JSX 
// ✅ Link من react-router-dom: تنقّل فوري بلا إعادة تحميل
import { Link } from 'react-router-dom';

function Navbar() {
  return (
    <nav>
      <Link to="/">الرئيسية</Link>
      <Link to="/about">من نحن</Link>
      <Link to="/contact">تواصل معنا</Link>
    </nav>
  );
}
4.2

NavLink: تنسيق الرابط النشط تلقائياً

JSX 
import { NavLink } from 'react-router-dom';

function Navbar() {
  const linkClass = ({ isActive }) => (isActive ? 'link-active' : '');

  return (
    <nav>
      <NavLink to="/" className={linkClass}>الرئيسية</NavLink>
      <NavLink to="/about" className={linkClass}>من نحن</NavLink>
    </nav>
  );
}
// isActive تُصبح true تلقائياً عندما يكون هذا الرابط مطابقاً للمسار الحالي

الجزء الخامس: Route Parameters وuseParams

5.1

تعريف مسار ديناميكي

JSX 
<Routes>
  <Route path="/products" element={<ProductsList />} />
  <Route path="/products/:id" element={<ProductDetails />} /> {/* :id معامل ديناميكي */}
</Routes>
5.2

قراءة المعامل بـ useParams

JSX 
// src/pages/ProductDetails.jsx
import { useParams } from 'react-router-dom';

function ProductDetails() {
  const { id } = useParams(); // يقرأ القيمة الفعلية من عنوان URL الحالي

  return <h1>تفاصيل المنتج رقم: {id}</h1>;
}
// زيارة /products/42 في المتصفح → useParams() تُعيد { id: '42' }
🚀

الجزء السادس: useNavigate — التنقل البرمجي

أحياناً نحتاج التنقّل نتيجة حدث برمجي (بعد إرسال نموذج، بعد تسجيل دخول ناجح) لا نتيجة ضغط رابط مباشرة.

6.1

مثال: التنقل بعد تسجيل الدخول

JSX 
import { useNavigate } from 'react-router-dom';

function LoginForm() {
  const navigate = useNavigate();

  function handleSubmit(e) {
    e.preventDefault();
    // ... تحقّق من بيانات الدخول هنا
    navigate('/dashboard'); // تنقّل برمجي بعد نجاح تسجيل الدخول
  }

  return (
    <form onSubmit={handleSubmit}>
      {/* حقول النموذج */}
      <button type="submit">دخول</button>
    </form>
  );
}

الجزء السابع: المشروع — موقع متعدد الصفحات

موقع متجر بسيط: صفحة رئيسية، قائمة منتجات، صفحة تفاصيل منتج بمعامل ديناميكي، وصفحة 404.

src/App.jsx

JSX 
import { Routes, Route } from 'react-router-dom';
import Navbar from './components/Navbar';
import Home from './pages/Home';
import Products from './pages/Products';
import ProductDetails from './pages/ProductDetails';
import NotFound from './pages/NotFound';

function App() {
  return (
    <>
      <Navbar />
      <Routes>
        <Route path="/" element={<Home />} />
        <Route path="/products" element={<Products />} />
        <Route path="/products/:id" element={<ProductDetails />} />
        <Route path="*" element={<NotFound />} />
      </Routes>
    </>
  );
}

export default App;

src/components/Navbar.jsx

JSX 
import { NavLink } from 'react-router-dom';

function Navbar() {
  const linkClass = ({ isActive }) => (isActive ? 'nav-link active' : 'nav-link');

  return (
    <nav className="navbar">
      <NavLink to="/" className={linkClass}>الرئيسية</NavLink>
      <NavLink to="/products" className={linkClass}>المنتجات</NavLink>
    </nav>
  );
}

export default Navbar;

src/pages/Products.jsx وProductDetails.jsx

JSX 
// src/pages/Products.jsx
import { Link } from 'react-router-dom';

const products = [
  { id: 1, name: 'لابتوب' },
  { id: 2, name: 'هاتف ذكي' },
  { id: 3, name: 'سماعات' },
];

function Products() {
  return (
    <div>
      <h1>🛍️ المنتجات</h1>
      <ul>
        {products.map((p) => (
          <li key={p.id}>
            <Link to={`/products/${p.id}`}>{p.name}</Link>
          </li>
        ))}
      </ul>
    </div>
  );
}

export default Products;

// src/pages/ProductDetails.jsx
import { useParams, Link } from 'react-router-dom';

function ProductDetails() {
  const { id } = useParams();

  return (
    <div>
      <h1>تفاصيل المنتج #{id}</h1>
      <Link to="/products">⬅ العودة للمنتجات</Link>
    </div>
  );
}

export default ProductDetails;

الجزء الثامن: تمارين الجلسة الثانية والعشرين

✏️ تمرين 1 — مسارات أساسية15 دقيقة

  • أنشئ 3 صفحات: الرئيسية، من نحن، تواصل معنا
  • Navbar بروابط Link تنقل بينها بلا إعادة تحميل
  • صفحة 404 تظهر لأي مسار غير معرَّف

✏️ تمرين 2 — NavLink ومعاملات المسار20 دقيقة

  • صفحة "المدونة" بقائمة مقالات (عنوان + id لكل مقال)
  • كل مقال رابط Link لمسار /blog/:id
  • صفحة تفاصيل المقال تعرض العنوان المطابق للمعامل عبر useParams
  • استخدم NavLink في شريط التنقل مع تنسيق واضح للرابط النشط

✏️ تمرين 3 — useNavigate عملي25 دقيقة

  • نموذج بسيط (اسم + بريد إلكتروني)
  • عند الإرسال: تحقّق بسيط من الحقول، ثم انتقل تلقائياً لصفحة "/thank-you" بـ useNavigate
  • صفحة الشكر تعرض رسالة ترحيبية وزر "العودة للرئيسية"

الواجب: مدونة شخصية متعددة الصفحات

◆ الصفحات المطلوبة:

  • الرئيسية: نبذة قصيرة + آخر 3 مقالات

  • من أنا: نبذة عن الكاتب
  • تواصل: نموذج بسيط ينتقل لصفحة شكر بعد الإرسال (useNavigate)
  • صفحة 404 مخصَّصة

◆ متطلبات تقنية:

  • Navbar بروابط NavLink مع تنسيق واضح للصفحة الحالية
  • استخدام useParams لصفحة المقال الفردي
  • تنظيم الصفحات في مجلد src/pages/ والمكوّنات المشتركة في src/components/
