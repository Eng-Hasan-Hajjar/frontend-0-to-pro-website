# Front-End Master Book — من HTML إلى JavaScript
موقع تفاعلي واحد متكامل لتعليم تطوير الويب من الصفر باللغة العربية، بديل عن عروض PowerPoint — إعداد **م. حسن حجار**.

## التشغيل
افتح `index.html` مباشرة في أي متصفح حديث. لأفضل تجربة مع المعاينة الحيّة وتشغيل الأكواد، شغّل خادماً محلياً بسيطاً بدل الفتح المباشر من القرص:
```bash
npx serve .
# أو
python3 -m http.server 8000
```

## ملاحظة عن أكواد React (JSX)
أكواد React تحتوي JSX (خليط HTML داخل JavaScript) أو استخدام Hooks (مثل useState) — كلاهما لا يمكن تشغيله مباشرة عبر `new Function()` (يحتاج بيئة React كاملة). تكتشف الأداة تلقائياً هذه الحالات وتُصنِّفها:
- **JSX**: كتلة تحتوي وسوماً فعلية (`<div>`, `<Component />`)
- **React**: كتلة تستخدم Hook (مثل `useState(`) بلا وسوم JSX مباشرة
- كلتاهما تُعرَضان **بدون زر تشغيل** (لتفادي رسالة خطأ مضلِّلة)، بينما يبقى الزر يعمل بشكل طبيعي على كود JavaScript الخالص القابل للتنفيذ فعلاً.

## آليتا التفاعل (مهم لفهم الكود)
هذا الموقع يخدم نوعين مختلفين من المحتوى، ولكل منهما آلية تفاعل مناسبة، وكلتاهما موجودتان معاً في `script.js`:
- **جلسات HTML/CSS (01–08)**: زر **"👁 معاينة حيّة"** يعرض الكود مُصيَّراً فعلياً داخل `iframe` (`initPreviewButtons`, `buildPreviewDoc`).
- **جلسات JavaScript (09–16)**: زر **"▶ جرّب الكود"** ينفّذ الكود في بيئة معزولة ويعرض ناتج `console.log` (`runSandboxed`).

## ملاحظة تصحيح مهمة
كان هذا المستودع يحتوي سابقاً على ملفات المرحلتين مجتمعة، لكن بنسخة واحدة فقط من `script.js`/`index.html` (نسخة HTML/CSS) — ما جعل كل أزرار "جرّب الكود" في جلسات JavaScript (09–16) معطّلة تماماً بلا أي خطأ ظاهر، ومنع ظهور جلسات JavaScript في القائمة الرئيسية والشريط الجانبي. تم إصلاح ذلك ببناء `script.js` و`index.html` موحَّدين يدعمان الآليتين معاً، وبإعادة توليد الجلسات الـ16 بترقيم تنقّل متصل.
